## Prompt 库现已在 Opik 2.0 中可用

Prompt 库现在是 Opik 2.0 UI 的一部分，可从项目侧边栏的 **Prompt library** 访问。与此同时，prompt 版本获得了一流的环境支持——你可以将版本标记为 `production` 或 `staging`，并从 SDK 中按名称检索，无需在应用代码中追踪版本号。

**新增内容：**

- **侧边栏中的 Prompt 库** — Prompt 库现在是新 Opik 2.0 界面中每个项目的顶级部分
- **按环境获取** — `client.get_prompt(name, environment="production")` 返回当前标记为 production 的版本；`version` 和 `environment` 互斥，同时传递两者会引发明确错误
- **分配环境** — `client.set_prompt_environments(name, ["production", "staging"])` 替换版本上的完整环境集；同一环境会自动从之前持有它的版本移走
- **创建时标记** — `client.create_prompt(name, content="...", environments=["staging"])` 和 `client.create_chat_prompt(...)` 直接接受 `environments`
- **TypeScript 对等** — `setPromptEnvironments`、`getPrompt({ environment })` 和 `createPrompt({ environments })` 与 Python API 镜像
- **顺序版本号** — prompt 版本在 UI 和 API 中现在显示为 `v1`、`v2`、`v3`，而不是原始提交哈希
- **环境徽章无处不在** — 分配的环境出现在 prompt 库、历史时间线、差异视图和 Playground 中每个版本引用旁边
- **术语更新** — 整个 prompt UI 中"commit"已替换为"version"

```python
# 创建时标记版本
prompt = client.create_prompt("system-prompt", content="...", environments=["staging"])

# 按环境检索——无需硬编码版本号
production_prompt = client.get_prompt("system-prompt", environment="production")

# 将特定版本提升到 production
client.set_prompt_environments("system-prompt", ["production"], version="v3")
```

## 日志视图中的简化过滤器

Traces、Spans 和对话线程标签页现在有重新设计的过滤栏，让你更快地缩小查找范围。过滤器作为芯片直接出现在工具栏中——选择字段，设置值，表格即时更新。常用过滤器可以**固定**到栏上，始终一键可达，过滤状态保存在 URL 中，你可以与队友分享精确的过滤视图。

## Bug 修复和改进

- **测试套件断言：子 span 检查** — 评估器 LLM 现在可以发出 `get_trace_spans` 和 `read` 工具调用，在评估期间检查中间 spans，支持关于工具使用、模型选择和复杂 Agents 内部每 span 错误的正确性检查
- **Google ADK 集成：图像在 trace 附件中渲染** — ADK 发送的 URL 安全 base64 图像数据自动标准化为标准 base64；PNG、JPEG、GIF 和 WebP 附件都能正确渲染
- **优化试验页面：显示所有组成实验** — 属于多项目优化的实验现在从试验页面可见，无论用户当前从哪个项目查看优化
- **错误率 KPI：现在显示百分比** — 错误率仪表板卡片之前显示原始事件计数；现在显示为百分比
- **标注队列：内联显示 trace 日志** — trace 日志条目在标注队列页面上内联渲染，无需导航离开
- **在线评估规则：ClassCastException 已解决** — 包含过滤器的对话线程级别规则在某些配置下不再抛出 `ClassCastException`
- **附件：处理 data URI 前缀** — 包含 `data:<type>;base64,` 前缀的 base64 附件负载现在在 SDK 和前端中都能正确剥离
- **SDK：保留内置环境颜色** — 具有保留名称的工作区环境在更新或同步后保留其指定颜色
- **`opik migrate`：清晰报告跳过的项** — 迁移命令现在报告每个跳过项的原因、计数和示例源 ID，并以代码 1 退出，使 CI 流水线能检测不完整的迁移
- **千帆集成文档** — 千帆 LLM Provider 集成现在有专门的文档页面
