以下是自上次发布以来我们做出的最相关的改进：

## 优化工作室

我们很高兴推出**优化工作室**——一种无需编写代码即可改进 prompts 的强大新方式。提供一个 prompt，定义什么是"好"的标准，Opik 会测试变体以找到一个你可以放心发布的更好版本。

<video src="https://files.buildwithfern.com/https://opik.docs.buildwithfern.com/docs/opik/2026-01-27T08:27:40.904Z/img/agent_optimization/optimization_studio_walkthrough.mp4" width="854" height="480" autoPlay muted loop playsInline preload="auto" />

**新增内容：**

- **无代码 prompt 优化** - 优化工作室帮助你直接从 Opik UI 改进 prompts。你看到的是分数和示例，而不仅仅是直觉，缩短了从想法到证据的循环
- **算法选择** - 选择 Opik 如何搜索更好的 prompts：**GEPA** 适合单轮 prompts 和快速改进，而 **HRPO** 在你需要深入分析 prompt 失败原因时更合适
- **灵活的指标** - 定义 Opik 应如何对每个 prompt 变体评分。当你有唯一正确答案时使用 **Equals** 进行严格匹配，当答案可变且你想让模型评分质量时使用 **G-Eval**
- **可视化进度追踪** - 通过实时进度图表监控优化运行，显示当前最佳分数和每次试验的结果
- **试验比较** - 试验标签页让你并排比较 prompt 变体和分数，并能深入查看单个评估项
- **重新运行和比较** - 轻松重新运行相同设置、取消运行以更改输入，或选择多个运行来比较结果

对于偏好编程工作流的团队，我们还发布了 **Opik Optimizer SDK v3**，具有改进的算法、更好的性能和更直观的 API。

## 仪表板改进

我们增强了仪表板，新增组件和可视化功能，帮助你更有效地追踪和比较实验。

**新增内容：**

- **实验排行榜组件** - 新的排行榜组件让你直接从仪表板一目了然地排名和比较实验，更容易识别最佳性能配置
- **指标组件的分组功能** - 项目指标组件现在支持分组，允许你以更有意义的方式切片和分析指标数据
- **Span 级别指标图表** - 新图表提供了 span 级别指标的可见性，让你更深入地了解 traces 中单个组件的性能

## 视频生成支持

我们添加了对最新视频生成模型的支持，使你能够追踪和记录 AI 应用的视频输出。

**新增内容：**

- **OpenAI SORA 集成** - 直接在 Opik 中记录和追踪 OpenAI SORA 模型的视频生成输出
- **Google Veo 集成** - 完全支持 Google Veo 视频生成 API，包括自动记录视频输出和元数据

## 实验管理

我们通过新的过滤和标签功能使组织和导航实验更加容易。

**改进内容：**

- **实验视图中的项目列** - 实验现在直接在列表视图中显示其关联的项目，让你一目了然地了解上下文
- **项目过滤和分组** - 按项目过滤和分组实验，在大型实验集合中快速找到你要找的内容
- **实验标签** - 标签现在在实验页面上渲染，帮助你更轻松地分类和识别实验

## UI/UX 改进

我们进行了多项改进，使你的日常工作流更加顺畅。

**改进内容：**

- **时间格式设置** - 自定义整个 UI 中时间戳的显示方式，以匹配你偏好的格式
- **在线评分规则默认值** - 在线评分规则中的输入和输出字段现在预填充了合理的默认值，减少设置时间
- **数据集项导航** - 实验项视图中的导航标签现在直接链接到关联的数据集项，便于数据探索
- **标注队列审查** - 你现在可以审查已完成的标注队列，便于审计和验证标注工作

## SDK 和集成

我们通过更好的追踪和性能指标改进了 SDK 集成。

**改进内容：**

- **Vercel AI SDK 对话线程支持** - Vercel AI SDK 集成的对话线程 ID 支持实现了多轮交互的更好对话追踪
- **ADK 分布式追踪** - 在 ADK 集成中添加了分布式追踪 headers 支持，改善分布式系统的可观测性
- **首个 Token 响应时间（TTFT）** - ADK 集成现在捕获 TTFT 指标，为你提供流式应用响应延迟的可见性
