以下是自上次发布以来我们做出的最相关的改进：

## 自定义仪表板（Beta）

自定义仪表板现已上线！

<Frame>
  <img src="/img/production/dashboard_example.png" alt="自定义仪表板示例，展示可自定义组件用于追踪跨项目和实验的指标" />
</Frame>

我们的新仪表板引擎让你构建完全可自定义的视图，追踪从 token 使用和成本到延迟、跨项目和实验的质量等一切内容。

**在哪里找到它们？**

仪表板在 Opik 内的三个地方可用：

- **仪表板页面** - 从侧边栏创建和管理所有仪表板
- **项目页面** - 在仪表板标签页下查看项目特定的指标
- **实验比较页面** - 可视化和比较实验结果

**内置模板快速开始**

我们提供零设置的预构建模板仪表板，包括性能概览、实验洞察和项目运营指标。

模板完全可编辑，自定义后可保存为新仪表板。

**灵活的组件**

仪表板支持多种组件类型：

- **项目指标**（项目数据的时间序列和柱状图）
- **项目统计**（KPI 数字卡片）
- **实验指标**（实验数据的折线图、柱状图、雷达图）
- **Markdown**（备注、文档、上下文）

组件支持过滤、分组、调整大小、拖放布局和全局日期范围控制。

👉 [文档](/v1/production/dashboards)

## 改进的评估能力

**Span 级别指标**

Span 级别指标已在 Opik 中正式上线，支持 LLMaaJ 和基于代码的指标！

团队现在可以轻松地以精确的全精度评估其 Agent 流程中特定步骤的质量。
不再仅评估最终输出或顶层 trace，你可以将指标直接附加到单个调用 span 或 Agent 轨迹的片段。

这解锁了显著更细粒度的可见性和控制。例如：

- 为面向任务或使用工具的 Agent 内部的关键决策点评分
- 独立衡量子任务的性能以定位瓶颈或回归
- 跨运行、实验或版本逐步比较 Agent 行为

**在线评分中访问完整树、子树或叶节点的新支持**

此更新增强了在线评分引擎，支持在 LLM-as-Judge 和基于代码的评估器中引用整个根对象（输入、输出、元数据），而不仅仅是其中的嵌套字段。

在线评分之前仅暴露 LLM 结构化输出的叶级别值。通过此更新，Opik 现在支持渲染任何子树：从单个节点到整个嵌套结构。

## Prompt 版本管理的标签支持和元数据过滤

你现在可以为单个 prompt 版本打标签（不仅仅是 prompt 本身）！

这提供了一种干净、直观的方式来标记最佳性能版本、管理生命周期，并将版本选择集成到 Agent 部署中。

## 更多多模态支持：现在支持音频！

你现在可以在 Playground 和在线评估中将音频作为 prompts 的一部分传递，用于高级多模态场景。

## 更多洞察！

**对话线程级别洞察**

在对话线程表中添加了新指标，包含对话线程级别指标和统计信息，为用户提供了关于完整多轮 Agent 交互的聚合洞察：

- 时长百分位数（p50、p90、p99）和平均值
- Token 使用统计（总数、prompt、completion tokens）
- 成本指标和聚合
- 还添加了按项目、时间范围和自定义过滤器的过滤支持

**实验洞察**

在实验项的表头中添加了额外的聚合方法。

此新版本在实验项表头中为所有数值指标添加了百分位数聚合方法（p50、p90、p99），将之前用于时长的模式扩展到成本、反馈评分和总 token。

## 集成

**Playground 和在线评分中支持 GPT-5.2**

在 Playground 和在线评分功能中为 OpenAI 和 OpenRouter Provider 添加了对 GPT 5.2 模型的完整支持。

**Harbor 集成**

为 Harbor 添加了全面的 Opik 集成，Harbor 是一个用于自主 LLM Agents 的基准评估框架。该集成为 Agent 基准评估（SWE-bench、LiveCodeBench、Terminal-Bench 等）提供了可观测性。
