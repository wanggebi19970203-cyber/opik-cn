以下是自上次发布以来我们做出的最相关的改进：

## 多值反馈评分和标注队列
我们很高兴地宣布评估和标注功能的重大改进！

### 新增内容

**1. 多值反馈评分**
多个用户现在可以独立对同一个 trace 或对话线程进行评分。不再互相覆盖——每位评审者的观点都被保留并在产品中可见。这使得评估过程中能够建立更丰富、更可靠的共识。

**2. 标注队列**
创建需要专家审查的 traces 或对话线程队列。通过简单链接与领域专家分享。系统化地组织工作、追踪进度，并大规模收集结构化和非结构化反馈。

**3. 简化的标注体验**
专为非技术评审者设计的简洁、专注的 UI。支持清晰的说明、预定义的反馈指标和进度指示器。轻量且无干扰，让领域专家可以专注于提供高质量反馈。

## Opik 优化器 - GEPA 算法和 MCP 工具优化

### 新增内容

**1. GEPA（遗传-帕累托）支持**
[GEPA](https://github.com/gepa-ai/gepa/) 是斯坦福推出的新一代 prompt 优化算法。这增强了我们现有的优化器，为用户提供了更多选择。

**2. MCP 工具调用优化**
调优 MCP 服务器（LLM 使用的外部工具）的能力。我们的解决方案使用现有算法（MetaPrompter）来调优 LLM 与 MCP 工具的交互方式。最终输出是一个新的工具签名，你可以将其提交回代码中。
