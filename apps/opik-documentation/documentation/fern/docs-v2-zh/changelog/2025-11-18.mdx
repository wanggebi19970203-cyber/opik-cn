以下是自上次发布以来我们做出的最相关的改进：

## 更多指标！

我们发布了 **37 个新内置指标**，更快更可靠的 LLM 评判，以及健壮性修复。

**新增指标** - 我们扩展了评估指标库，提供了一套全面的开箱即用指标，包括：
- **经典 NLP 启发式方法** - BERTScore、情感分析、偏见检测、对话漂移等
- **轻量级启发式方法** - 快速、非 LLM 的指标，非常适合 CI/CD 流水线和大规模评估
- **LLM-as-a-Judge 预设** - 更多开箱即用的预设，无需自定义配置即可使用

**LLM-as-a-Judge 和 G-Eval 改进**：
- **兼容新模型** - 现在可与最新模型版本无缝配合
- **更快的默认评判器** - 默认评判器现在是 `gpt-5-nano`，评估更快更准确
- **LLM 陪审团支持** - 跨多个模型/评判器聚合分数为单一集成分数，实现更可靠的评估

**增强的预处理**：
- **改进的英文文本处理** - 更好地处理英文文本以减少误报
- **更好的 emoji 处理** - 增强的 emoji 处理，实现更准确的评估

**健壮性改进**：
- **自动重试** - LLM 评判器在瞬态故障时会重试，避免不稳定的测试结果
- **更可靠的评估运行** - CI 和实验的评估运行更快更一致

## 匿名化器 - PII 信息脱敏

我们添加了在将数据发送到 Opik 之前**支持 PII（个人可识别信息）脱敏**的功能。这有助于你保护敏感信息，同时仍能获得所需的可观测性洞察。

通过匿名化器，你可以：
- 在 traces 和 spans 发送到 Opik 之前**自动脱敏 PII**
- **配置自定义匿名化规则**以匹配你特定的隐私要求
- **保持合规**，遵守数据保护法规
- **保护敏感数据**而不损失可观测性

👉 阅读完整文档：[匿名化器](/v1/production/anonymizers)

## 新告警类型

我们扩展了告警功能，新增告警类型和改进功能：

- **实验完成告警** - 实验完成时收到通知，以便立即审查结果或触发 CI/CD 流水线。
- **成本告警** - 为成本指标设置阈值，当支出超过限制时接收告警
- **延迟告警** - 监控响应时间，当延迟超过配置的阈值时收到通知

这些新的告警类型帮助你掌控 LLM 应用的性能和成本，实现主动监控和更快的问题响应。

## 多模态支持

我们显著增强了平台的多模态能力：

- **视频 LLM-as-a-Judge** - 添加了视频 LLM-as-a-Judge 支持，能够评估 traces 中的视频内容
- **视频成本追踪** - 添加了视频模型的成本追踪，监控视频处理操作的支出

- **LLM-as-a-Judge 中的图像支持** - Python 和 TypeScript SDK 现在都支持 LLM-as-a-Judge 评估中的图像处理，允许你评估包含图像的 traces

这些增强功能使构建和评估处理图像和视频内容的多模态应用变得更加容易。

## 自定义 AI Provider

我们改进了对自定义 AI Provider 的支持，增强了配置选项：

- **多个自定义 Provider** - 设置多个自定义 AI Provider，用于 Playground 和在线评分
- **自定义 Headers 支持** - 为你的自定义 Provider 配置自定义 headers，为你连接企业 AI 服务提供更多灵活性


## 增强的评估和可观测性

我们添加了多项改进，使评估和可观测性更加强大：

- **数据集中的 Trace 和 Span 元数据** - 能够将 trace 和 span 元数据添加到数据集中，用于高级 Agent 评估，支持更复杂的评估工作流
- **Token 分解显示** - 在 trace 视图中显示 token 分解（输入/输出），为你提供每个 span 和 trace 的详细 token 使用可见性
- **二进制（布尔）反馈评分** - 新增对二进制（布尔）反馈评分的支持，允许你捕获简单的通过/失败评估


## 用户体验改进

我们在平台上进行了多项用户体验增强：

- **改进的美化模式** - 增强了 traces、对话线程和标注队列的美化模式，使数据更易阅读和理解
- **Traces、对话线程和 Spans 的日期过滤** - 添加了日期过滤功能，允许你在分析数据时聚焦于特定时间范围
- **新的优化运行部分** - 在主页添加了新的优化运行部分，让你快速访问优化结果
- **Comet 调试器模式** - 添加了 Comet 调试器模式，包含应用版本和连接状态，帮助你排查问题并了解应用的连接状态。在此[了解更多](/v1/faq#using-comet-debugger-mode-uibrowser)
