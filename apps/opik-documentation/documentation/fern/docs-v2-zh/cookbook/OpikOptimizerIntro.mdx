## 数据集

在此实验中，我们将使用 **HotPotQA** 数据集。该数据集被设计为对常规 LLM 来说难以处理。该数据集被称为"**多跳**"数据集，因为回答问题涉及多个推理步骤和多个工具调用，LLM 需要推断关系、组合信息或基于组合上下文得出结论。

示例：

> "与加利福尼亚接壤的州的首都是什么？"

您需要找到与加利福尼亚接壤的州，然后查找每个州的首府。

该数据集包含约 113,000 个此类众包问题，这些问题被构建为需要两篇维基百科文章的引言段落来回答。

**注意：**"HotPot"这个名字来自于作者构思该数据集的餐厅。


```python
from opik_optimizer.demo import get_or_create_dataset

opik_dataset = get_or_create_dataset("hotpot-300")
```

## 指标

为优化选择一个好的指标很棘手。对于这些示例，我们将选择一个能够展示改进并提供分数梯度的指标。不过一般来说，这个指标不是优化运行的最佳选择。

我们将使用"编辑距离"又名"Levenshtein 距离"：


```python
from opik.evaluation.metrics import LevenshteinRatio
metric = LevenshteinRatio(project_name=project_name)
```

该指标需要两样东西：LLM 的 `output` 和 `reference`（真实值）：


```python
metric.score("Hello", "Hello")
```


```python
metric.score("Hello!", "Hello")
```

"Hello!"和"Hello"之间的编辑距离为 1。以下是 .91 的计算方式：


```python
edit_distance = 1

1 - edit_distance / (len("Hello1") + len("Hello"))

```

更多信息请参阅：[Levenshtein 距离](https://en.wikipedia.org/wiki/Levenshtein_distance)

## 配置

要创建使用 Opik Agent 优化器所需的配置，您需要三样东西：

1. 初始提示
2. MetricConfig
3. TaskConfig

我们将从一个相当糟糕的提示开始...这样我们就可以优化它！


```python
initial_prompt = "Provide an answer to the question"
```

两个配置：


```python
from opik_optimizer import (
    MetricConfig,
    TaskConfig,
    from_llm_response_text,
    from_dataset_field,
)

metric_config = MetricConfig(
    metric=LevenshteinRatio(project_name=project_name),
    inputs={
        "output": from_llm_response_text(),
        "reference": from_dataset_field(name="answer"),
    },
)

task_config = TaskConfig(
    instruction_prompt=initial_prompt,
    input_dataset_fields=["question"],
    output_dataset_field="answer",
    use_chat_prompt=True,
)
```

如您所见，`MetricConfig` 由我们选择的指标组成。此外，我们需要知道输入是什么。这里的输入实际上是 LLM 的输出。

我们需要两个指标输入：
1. LLM 产生的输出（使用特殊名称）
2. 正确答案（由数据库项目"answer"提供）

`TaskConfig` 定义了如何处理提示。我们需要初始提示，以及数据集的输入和输出。

在这种情况下，我们将使用 chat_prompt 格式作为结果。

## FewShotBayesianOptimizer

FewShotBayesianOptimizer 名称表示两件事：

1. 它将生成聊天提示，或如幻灯片中描述的 FewShot 示例。
2. 其次，它描述了如何搜索这些 FewShot 示例的最佳集合。

要使用此优化器，我们导入它并创建一个实例，传入项目名称和模型参数：


```python
from opik_optimizer import (
    FewShotBayesianOptimizer,
)

optimizer = FewShotBayesianOptimizer(
    project_name=project_name,
    model="openai/gpt-4o-mini",
    temperature=0.1,
    max_tokens=5000,
)
```

### 基线

在优化此提示（"Provide an answer to the question"）之前，让我们看看这个简单提示本身在数据集上的表现：


```python
score = optimizer.evaluate_prompt(
    dataset=opik_dataset,
    metric_config=metric_config,
    task_config=task_config,
    prompt=initial_prompt,
    n_samples=100,
)
score
```

在我的运行中，它得分约 16% 正确。[我说"百分比正确"，但由于我们使用的是编辑距离，这并不完全准确。但我们可以这样理解。] 您的结果可能有所不同，但可能在 10% 到 20% 正确之间。

好的，让我们优化那个提示！


```python
result = optimizer.optimize_prompt(
    opik_dataset,
    metric_config,
    task_config,
    n_trials=3,
    n_samples=50
)
```


```python
result.display()
```

优化器做得很好！

正确百分比从约 15% 提高到约 50%。

我们发现了什么？结果是一系列消息：


```python
result.details["chat_messages"]
```

## 使用优化后的提示

我们如何使用优化后的结果？


一旦我们有了"chat_messages"，我们可以执行以下操作：


```python
from litellm.integrations.opik.opik import OpikLogger
import litellm
opik_logger = OpikLogger()
litellm.callbacks = [opik_logger]

def query(question, chat_messages):
    messages = chat_messages[:-1] # Cut off the last one
    # replace it with question in proper format:
    messages.append({'role': 'user', 'content': '{"question": "%s"}"}' % question)

    response = litellm.completion(
        model="gpt-4o-mini",
        temperature=0.1,
        max_tokens=5000,
        messages=messages,
    )
    return response.choices[0].message.content
```


```python
query("When was David Chalmers born?", result.details["chat_messages"])
```


```python
query("What weighs more: a baby elephant or an SUV?", result.details["chat_messages"])
```

如果它说"elephant"，那是不正确的！

我们需要使用带有工具的代理来获得更好的答案。
