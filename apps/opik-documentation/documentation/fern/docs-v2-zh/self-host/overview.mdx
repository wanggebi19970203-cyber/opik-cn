您可以通过 [Comet 的托管云服务](https://comet.com) 使用 Opik，也可以在您自己的基础设施上自托管 Opik。选择自托管 Opik 时，您可以访问所有 Opik 功能，包括追踪、评估等，但不包含用户管理功能。

如果选择自托管 Opik，您可以在两种部署选项中进行选择：

1. [本地安装](/self-host/local_deployment)：非常适合入门，但不适用于生产环境。
2. [Kubernetes 安装](/self-host/kubernetes)：生产就绪的 Opik 平台，运行在 Kubernetes 集群上。

## 快速开始

如果您想在本地试用 Opik，我们建议使用我们的本地安装脚本。假设您已安装 `git` 和 `docker`，只需一行代码即可开始：

<Tabs>
 <Tab value="Linux / Mac" title="Linux / Mac">
```bash
# 克隆 Opik 仓库
git clone https://github.com/comet-ml/opik.git

# 进入 opik 文件夹

cd opik

# 启动 Opik 平台

./opik.sh

```

</Tab>
<Tab value="Windows" title="Windows">

```powershell
# 克隆 Opik 仓库
git clone https://github.com/comet-ml/opik.git

# 进入 opik 文件夹
cd opik

# 启动 Opik 平台
powershell -ExecutionPolicy ByPass -c ".\opik.ps1"
```

</Tab>
</Tabs>

Opik 现在可以在以下地址访问

<a href="http://localhost:5173" target="_blank">
  http://localhost:5173
</a>
您本地机器记录的所有追踪数据都将记录到此本地 Opik 实例中。为了将追踪和其他数据记录到您的 Opik 实例，您需要确保 Opik Python SDK 配置为指向您刚刚启动的 Opik 服务器。您可以通过运行以下命令来完成配置：

```bash
# 配置 Python SDK 指向本地 Opik 平台
export OPIK_BASE_URL=http://localhost:5173/api
```

或在 Python 中：

```python
import os

os.environ["OPIK_BASE_URL"] = "http://localhost:5173/api"
```

要了解如何管理您的本地 Opik 部署，请参阅我们的[本地部署指南](/self-host/local_deployment)。

## SDK 版本兼容性

对于自托管部署，我们建议将 Opik 服务器和 SDK 保持在最新版本以获得最佳兼容性。Python SDK (`opik`)、TypeScript SDK (`opik`) 和 Opik Agent Optimizer SDK (`opik-optimizer`) 设计为与最新的 Opik 服务器版本配合使用。较旧的 SDK 版本可能不支持较新的服务器功能，较新的 SDK 版本可能需要最低服务器版本。请查看[更新日志](https://www.comet.com/docs/opik/changelog)了解特定版本的兼容性说明。
