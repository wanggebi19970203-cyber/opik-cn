Opik 使用基于角色的访问控制（RBAC）系统，允许你定义用户在工作区中可以执行的操作。本指南介绍角色和权限的工作方式、可用的默认角色，以及如何创建自定义角色。

## 组织角色与工作区角色

Opik 有两个层级的角色协同工作：

| 层级 | 用途 | 范围 | 示例 |
| --- | --- | --- | --- |
| **组织角色** | 控制对组织级功能的访问 | 整个组织 | 管理员、成员、只读成员 |
| **工作区角色** | 控制用户在工作区内的操作 | 单个工作区 | 管理、写入、标注、只读 |

用户的实际访问权限由其**组织角色和工作区角色共同决定**：

- 组织角色设定了用户在所有工作区中可获得的最高访问级别（例如，只读成员在整个组织中仅限只读访问）。
- 工作区角色决定了用户在其所属的每个工作区中可以执行的操作，但不能超过组织角色设定的上限。

## 组织角色

你组织中的每个用户都有一个唯一的组织角色：

| 角色 | 描述 |
| --- | --- |
| **管理员** | 拥有对[管理控制台](/administration/admin-dashboard/overview)和组织内所有工作区的完全访问权限。 |
| **成员** | 可以获得对其所加入工作区的完全访问权限。无法访问管理控制台。 |
| **只读成员** | 对所加入的工作区拥有只读访问权限。在任何工作区中都无法被授予写入权限。 |

新用户默认分配**成员**角色。组织管理员可以在管理控制台的[用户](/administration/admin-dashboard/users)页面更改用户的角色。

## 工作区角色

工作区角色控制用户在特定工作区中可以执行的操作。用户可以在不同的工作区中拥有不同的角色。

### 默认角色

这些角色在所有 Opik 组织中可用，作为自定义角色的基础：

| 角色 | 描述 |
| --- | --- |
| **管理** | 完全管理权限。管理成员、设置和所有资源。 |
| **写入** | 读写访问权限。创建项目、记录追踪、运行实验。 |
| **标注** | 标注访问权限。查看数据并添加标注/反馈。 |
| **只读** | 只读访问权限。查看所有数据但无法进行修改。 |

### 自定义角色

<Note>
自定义角色适用于企业版计划。[联系支持](https://www.comet.com/site/about-us/contact-us/)以启用此功能。
</Note>

创建自定义角色：

1. 进入 **管理控制台** > **角色与权限**。
2. 点击 **创建角色** 并配置权限。
3. 以现有角色为基础进行继承。

## 分配角色

根据角色类型，可以在不同的位置更新角色：

- **组织角色**可由组织管理员在 **管理控制台** > **[用户](/administration/admin-dashboard/users)** 中更新。
- **工作区角色**可由具有管理工作区角色的用户在 **配置** > **[成员](/administration/workspace-settings/workspace_members)** 中更新。
