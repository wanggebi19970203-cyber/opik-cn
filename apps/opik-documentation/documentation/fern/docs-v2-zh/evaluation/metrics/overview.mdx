# 概览

Opik 提供了一组内置评估指标，您可以自由组合使用这些指标来评估 LLM 的行为。这些指标分为两大类：

1. **启发式指标** — 基于规则、统计或经典 NLP 算法的确定性检查。
2. **LLM 作为评判者指标** — 将评分委托给 LLM，以便捕获语义、任务特定或对话级别的质量信号。

当您需要可复现的检查时（如精确匹配、正则表达式验证或与参考的相似度评分），启发式指标是理想选择。当您需要更丰富的定性反馈时（如幻觉检测、有用性、摘要质量、合规风险等），LLM 作为评判者指标更为适用。

## 自定义 LLM 作为评判者指标

默认情况下，Opik 使用 OpenAI 的 GPT-5-nano 作为 LLM 来评估其他 LLM 的输出。但您可以通过指定不同的 `model` 参数轻松切换到其他 LLM 提供商。

<CodeBlocks>
```python title="Python" language="python"
from opik.evaluation.metrics import Hallucination

metric = Hallucination(model="bedrock/anthropic.claude-3-sonnet-20240229-v1:0")

metric.score(
input="What is the capital of France?",
output="The capital of France is Paris. It is famous for its iconic Eiffel Tower and rich cultural heritage.",
)

````

```typescript title="TypeScript" language="typescript"
import { Hallucination } from 'opik';
import { openai } from '@ai-sdk/openai';

// Using model ID string (simplest approach)
const metric1 = new Hallucination({ model: 'gpt-4o' });
const metric2 = new Hallucination({ model: 'claude-3-5-sonnet-latest' });
const metric3 = new Hallucination({ model: 'gemini-2.0-flash' });

// With generation parameters (temperature, seed, maxTokens)
const metric4 = new Hallucination({
  model: 'gpt-4o',
  temperature: 0.3,
  seed: 42
});

// Using custom LanguageModel instance for provider-specific configuration
const customModel = openai('gpt-4o', {
  structuredOutputs: true
});
const metric5 = new Hallucination({ model: customModel });

// Score using the metric
await metric4.score({
    input: "What is the capital of France?",
    output: "The capital of France is Paris. It is famous for its iconic Eiffel Tower and rich cultural heritage.",
});
````

</CodeBlocks>

对于 **Python**，此功能基于 LiteLLM 框架。您可以在 [LiteLLM 提供商](https://docs.litellm.ai/docs/providers) 指南中找到支持的 LLM 提供商完整列表及其配置方法。

对于 **TypeScript**，SDK 与 Vercel AI SDK 集成。您可以使用模型 ID 字符串以简化配置，或使用 LanguageModel 实例进行高级配置。更多详情请参阅 [Models 文档](/reference/typescript-sdk/evaluation/models)。
