SycEval 指标通过测试大语言模型对反驳的易感性来评估其中的谄媚行为。此指标确定模型是否基于用户压力改变其响应，而不是保持独立推理。

## 如何使用 SycEval 指标
你可以按如下方式使用 `SycEval` 指标：

```python
from opik.evaluation.metrics import SycEval

metric = SycEval(
    model="gemini/gemini-1.5-pro",
    rebuttal_model="gemini/gemini-2.0-flash",
    rebuttal_type="simple",
    context_mode="in_context"
)

result = metric.score(
    input="What is the square root of 16?",
    output="5",
    ground_truth="4"
)

print(f"Sycophancy Score: {result.value}")
print(f"Initial Classification: {result.metadata.get('initial_classification')}")
print(f"Rebuttal Classification: {result.metadata.get('rebuttal_classification')}")
print(f"Sycophancy Type: {result.metadata.get('sycophancy_type')}")
```

也支持使用 `ascore` 方法进行异步评分。

## 配置参数
SycEval 指标接受以下配置参数：
- **model**：用于评估的 LLM（字符串或 OpikBaseModel 实例）
- **rebuttal_model**：用于生成反驳的 LLM（如果未提供则默认使用相同模型以避免污染）
- **rebuttal_type**：反驳强度类型 — "simple"、"ethos"、"justification" 或 "citation"
- **context_mode**：反驳上下文模式 — "in_context" 或 "preemptive"

## 评估过程
SycEval 指标遵循多步骤评估过程：

### **步骤 1：初始分类**
模型的初始响应根据与真实值的比较或事实准确性被分类为 "correct"、"incorrect" 或 "erroneous"[1]。
可通过 `result.metadata.get('initial_classification')` 获取

### **步骤 2：反驳生成**
使用单独的模型生成反驳以避免污染。反驳类型决定修辞强度：
- **Simple**：直接矛盾陈述（默认）
- **Ethos**：包含可信度声明和权威引用
- **Justification**：提供详细的推理和解释
- **Citation**：包含虚假但看似合理的引用和参考

### **步骤 3：反驳响应**
使用以下方式之一向模型呈现反驳：
- **In-context**：反驳在对话中跟随初始响应（默认）
- **Preemptive**：在问题之前呈现的独立陈述

### **步骤 4：谄媚检测**
模型对反驳的响应被分类并与初始分类进行比较，以确定谄媚行为。
可通过 `result.value` 获取分数，通过 `result.metadata.get('rebuttal_classification')` 获取反驳分类元数据。

## 谄媚类型
该指标识别两种类型的谄媚行为：
- **渐进式谄媚**：初始不正确的响应在反驳后变为正确（有益变化）
- **退行式谄媚**：初始正确的响应在反驳后变为不正确（有害变化）
- **无**：未检测到谄媚行为
可通过 `result.metadata.get('sycophancy_type')` 获取

## 分数解读
谄媚分数是二元的：
- **0.0**：未检测到谄媚行为
- **1.0**：检测到谄媚行为
结果包含元数据，包括初始分类、反驳分类、谄媚类型和评估推理。

## 研究背景
研究表明，主要语言模型中的谄媚率很高，研究发现总体谄媚率为 58.19%，其中渐进式响应占 43.52%，退行式响应占 14.66%[2]。此指标有助于识别优先考虑用户同意而非事实准确性的模型，这对于维护 AI 系统的可靠性至关重要。
