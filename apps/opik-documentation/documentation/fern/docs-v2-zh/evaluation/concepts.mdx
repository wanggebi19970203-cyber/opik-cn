Opik 提供两种互补的方法来评估您的 LLM 应用程序。了解何时使用哪种方法将帮助您构建稳健的评估策略。

## 测试套件 — 基于断言的测试

测试套件让您将预期行为定义为自然语言断言。LLM 评判器会根据智能体的输出检查每个断言，并报告通过/失败结果。

**适用于：**
- 测试特定行为（例如，"响应不包含幻觉"）
- 智能体输出的通过/失败验证
- 迭代提示词并比较版本
- 在更改后捕获回归问题

测试套件包含三个主要组件：

1. **测试项**：智能体的输入数据（例如，带有上下文的问题、用户场景）
2. **断言**：预期行为的自然语言描述，由 LLM 评判器检查（例如，"响应应简洁"）
3. **执行策略**：控制每个项目的运行次数以及必须通过的运行次数

断言可以在两个层级定义：
- **套件级断言**适用于每个测试项
- **项目级断言**仅适用于特定测试项，是在套件级断言基础上的补充

### 通过/失败逻辑

- 如果所有断言都通过，则一次**运行**通过
- 如果通过的运行次数满足 `pass_threshold`，则一个**项目**通过
- **通过率**是通过项目数与总项目数的比率

## 数据集与指标 — 量化评分

基于数据集的评估使用量化指标对智能体的输出进行评分。您定义一个测试用例数据集，针对这些用例运行智能体，并使用预构建或自定义指标对结果进行评分。

**适用于：**
- 使用通用指标（幻觉、相关性、连贯性）衡量多个追踪的质量
- 使用数值评分比较模型或提示词版本
- 使用上下文精确率/召回率指标评估 RAG 管道
- 跨实验构建排行榜

基于数据集的评估包含三个主要组件：

1. **数据集**：包含输入和可选预期输出的测试用例集合
2. **任务**：接收数据集项目并返回智能体输出的函数
3. **指标**：评估输出的评分函数（例如，`Hallucination`、`AnswerRelevance`、自定义指标）

每次评估运行会创建一个**实验** — 记录每个数据集项目、智能体的输出和指标评分。实验存储在 Opik 中，您可以进行并排比较。

## 两种方法的选择

| | 测试套件 | 数据集与指标 |
|---|---|---|
| **输出** | 每个断言的通过/失败 | 每个指标的数值评分 |
| **评估方法** | LLM 评判器检查自然语言断言 | 评分函数（基于 LLM 或启发式） |
| **适用于** | 行为测试、回归检查 | 质量衡量、基准测试 |
| **迭代方式** | 更新断言，重新运行套件 | 更新数据集或指标，重新运行实验 |

您可以同时使用两种方法。例如，在开发过程中使用测试套件验证特定行为，在 CI 中使用数据集与指标跟踪质量评分的变化趋势。
