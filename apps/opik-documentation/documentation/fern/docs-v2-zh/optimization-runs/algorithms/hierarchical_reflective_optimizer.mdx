`HRPO`（层次反射提示词优化器）使用层次根因分析来识别和解决提示词中的特定故障模式。它分析评估结果，识别故障中的模式，并生成针对性的改进以系统地解决每个故障模式。

<Note>
  在代码中，使用 `HRPO` 类（例如 `from opik_optimizer import HRPO`）。底层模块名称仍使用 `hierarchical_reflective_optimizer` 以保持向后兼容性。
</Note>

<Note>
  当您有一个复杂的提示词，并希望基于理解*为什么*它失败来进行优化时，`HRPO` 是理想的选择。与生成许多随机变体的优化器不同，此优化器系统地分析故障，识别根本原因，并进行精确的改进以解决每个特定问题。
</Note>

## 工作原理

HRPO（层次反射提示词优化器）由 Opik 团队开发，用于改进可能已经经过几轮手动提示词工程的提示词。它专注于识别提示词失败的原因，然后更新提示词以解决问题。

由于数据集可能很大，我们将分析分成批次并并行分析。然后我们综合所有批次的发现以识别提示词的核心问题。

## 快速开始

您可以使用 `HRPO` 来优化提示词：

```python maxLines=1000
from opik_optimizer import HRPO, ChatPrompt, datasets
from opik.evaluation.metrics.score_result import ScoreResult

# 1. Define your evaluation dataset
dataset = datasets.hotpot(count=300)  # or use your own dataset

# 2. Configure the evaluation metric (MUST return reasons!)
def answer_quality_metric(dataset_item, llm_output):
    reference = dataset_item.get("answer", "")

    # Your scoring logic
    is_correct = reference.lower() in llm_output.lower()
    score = 1.0 if is_correct else 0.0

    # IMPORTANT: Provide detailed reasoning
    if is_correct:
        reason = f"Output contains the correct answer: '{reference}'"
    else:
        reason = f"Output does not contain expected answer '{reference}'. Output was too vague or incorrect."

    return ScoreResult(
        name="answer_quality",
        value=score,
        reason=reason  # Critical for root cause analysis!
    )

# 3. Define your initial prompt
initial_prompt = ChatPrompt(
    project_name="reflective_optimization",
    messages=[
        {
            "role": "system",
            "content": "You are a helpful assistant that answers questions accurately."
        },
        {
            "role": "user",
            "content": "Question: {question}\n\nProvide a concise answer."
        }
    ]
)

# 4. Initialize HRPO
optimizer = HRPO(
    model="gpt-4o",
    n_threads=8,
    max_parallel_batches=5,
    seed=42,
    model_parameters={"temperature": 0.7}
)

# 5. Run the optimization
optimization_result = optimizer.optimize_prompt(
    prompt=initial_prompt,
    dataset=dataset,
    metric=answer_quality_metric,
    n_samples=100,
    max_trials=5,
    max_retries=2
)

# 6. View the results
optimization_result.display()
```

### 模型支持

使用 `HRPO` 时需要考虑两个模型：
- `HRPO.model`：用于根因分析和故障模式综合的模型。
- `ChatPrompt.model`：用于评估提示词的模型。

`model` 参数接受任何 LiteLLM 支持的模型字符串（例如，`"gpt-4o"`、`"azure/gpt-4"`、`"anthropic/claude-3-opus"`、`"gemini/gemini-1.5-pro"`）。您还可以使用 `model_parameters` 参数传入额外的模型参数：

```python
optimizer = HRPO(
    model="anthropic/claude-3-opus-20240229",
    model_parameters={
        "temperature": 0.7,
        "max_tokens": 4096
    }
)
```
