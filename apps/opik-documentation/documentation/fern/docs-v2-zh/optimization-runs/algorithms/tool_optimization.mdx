工具优化是一项专门功能，允许您优化使用外部工具和模型上下文协议（MCP）的提示词。此功能由所有优化器支持，**除了** `FewShotBayesianOptimizer`、`ParameterOptimizer` 和 `GepaOptimizer`。

## 什么是工具优化？

工具优化扩展了传统的提示词优化，以处理包含以下内容的提示词：

- **MCP 工具** - 用于外部集成的模型上下文协议工具
- **工具模式** - 结构化工具定义和参数
- **多步骤工作流** - 涉及多个工具的复杂代理工作流

工具优化**不会**更改工具名称或模式。它更新：
- 工具**描述**
- 工具**参数描述**

<Note>
  **重要区别：**许多优化器（包括 GEPA、MetaPrompt 等）可以优化**使用工具的代理** - 这意味着优化具有函数调用或外部工具访问权限的代理的提示词。然而，**真正的工具优化**（优化工具描述和参数描述）是您可以在支持的优化器中使用 `optimize_tools=True` 启用的单独功能。
</Note>

<Note>

**为什么工具优化很重要：**

传统的提示词优化专注于基于文本的提示词，但现代 AI 应用通常需要：

- 与外部 API 和服务集成
- 通过函数调用进行结构化数据处理
- 使用工具进行复杂的多步推理
- 基于上下文的动态工具选择

工具优化确保这些复杂的提示词可以像简单的文本提示词一样被改进。

</Note>

## 支持的工具类型

### 1. 代理函数调用（不是真正的工具优化）

许多优化器可以优化**使用函数调用的代理**，但这与真正的工具优化不同。以下是 GEPA 优化器的示例：

```python
from opik_optimizer import GepaOptimizer, ChatPrompt

# GEPA example: optimizing an agent with function calling
prompt = ChatPrompt(
    system="You are a helpful assistant. Use the search_wikipedia tool when needed.",
    user="{question}",
    tools=[
        {
            "type": "function",
            "function": {
                "name": "search_wikipedia",
                "description": "This function searches Wikipedia abstracts.",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "query": {
                            "type": "string",
                            "description": "The term or phrase to search for."
                        }
                    },
                    "required": ["query"]
                }
            }
        }
    ],
    function_map={
        "search_wikipedia": lambda query: search_wikipedia(query, use_api=True)
    }
)

# GEPA optimizes the agent's prompt, not the tools themselves
optimizer = GepaOptimizer(model="gpt-5-nano")
result = optimizer.optimize_prompt(prompt=prompt, dataset=dataset, metric=metric)
```

<Note>
  这是**代理优化**（优化使用工具的代理的提示词），而不是**工具优化**（优化工具本身）。
</Note>

### 2. MCP（模型上下文协议）工具

**真正的工具优化**可用于 MCP 工具和函数调用工具，适用于所有支持的优化器。MCP 工具为外部集成提供标准化接口：

```python
# MCP tool optimization example
# See scripts/litellm_metaprompt_context7_remote_example.py for a working example

from opik_optimizer import MetaPromptOptimizer

# MCP tools are configured as OpenAI-style entries (local or remote)
optimizer = MetaPromptOptimizer(model="gpt-5-nano")

# Any supported optimizer works here (e.g., HRPO, Evolutionary, MetaPrompt).
```

<Note>
MCP 工具优化：

    - 由所有优化器支持，**除了** FewShotBayesianOptimizer、ParameterOptimizer 和 GepaOptimizer
    - 工作示例可在 `scripts/litellm_metaprompt_context7_remote_example.py` 中找到

</Note>

## 工具优化如何工作

支持的优化器通过专门的优化过程处理启用了工具的提示词：

### 1. 工具感知分析

优化器分析：

- **工具模式** - 理解可用函数及其参数
- **工具使用模式** - 工具在提示词中通常如何被调用
- **工具依赖关系** - 不同工具之间的关系
- **上下文要求** - 工具有效运行所需的信息

### 2. 提示词-工具集成优化

优化器可以改进：

- **工具选择逻辑** - 何时使用哪个工具的更好指令
- **参数格式化** - 如何结构化工具输入的更清晰指导
- **错误处理** - 处理工具故障或边缘情况的指令
- **工具链接** - 优化多步骤工具工作流

### 3. 上下文增强

工具优化还改进：

- **输入验证** - 验证工具输入的更好提示词
- **输出处理** - 处理工具输出的指令
- **回退策略** - 工具不可用时的替代方法

## 示例：优化研究助手

让我们看看工具优化如何与使用多个工具的研究助手一起工作：

```python
from opik_optimizer import MetaPromptOptimizer, ChatPrompt
from opik.evaluation.metrics import LevenshteinRatio

# Define a research assistant prompt with tools
research_prompt = ChatPrompt(
    messages=[
        {
            "role": "system",
            "content": """You are a research assistant. When given a research question:
1. Search for relevant information using the search tool
2. Analyze the results using the analysis tool
3. Provide a comprehensive answer based on your findings

Always cite your sources and be thorough in your research."""
        },
        {
            "role": "user",
            "content": "{research_question}"
        }
    ],
    tools=[
        {
            "type": "function",
            "function": {
                "name": "search_academic_database",
                "description": "Search academic papers and research",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "query": {"type": "string"},
                        "year_range": {"type": "string"},
                        "max_results": {"type": "integer"}
                    }
                }
            }
        },
        {
            "type": "function",
            "function": {
                "name": "analyze_text",
                "description": "Analyze and summarize text content",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "text": {"type": "string"},
                        "analysis_type": {"type": "string"}
                    }
                }
            }
        }
    ]
)

# Initialize the optimizer
optimizer = MetaPromptOptimizer(
    model="openai/gpt-5-nano"
)

# Define evaluation metric
def research_quality_metric(dataset_item, llm_output):
    return LevenshteinRatio().score(
        reference=dataset_item['expected_answer'],
        output=llm_output
    )

# Run optimization
result = optimizer.optimize_prompt(
    prompt=research_prompt,
    dataset=research_dataset,
    metric=research_quality_metric,
    n_samples=100,
    max_trials=5
)

print("Optimized prompt with tools:")
print(result.prompt)
```

## 工具优化最佳实践

### 1. 工具模式设计

- **清晰的描述** - 为每个工具提供详细的描述
- **全面的参数** - 包含所有必要参数及其类型
- **使用示例** - 在有帮助时在工具描述中添加示例
- **错误处理** - 定义预期的错误条件和响应

### 2. 提示词结构

- **工具介绍** - 向模型清晰地解释可用工具
- **使用指南** - 提供关于何时和如何使用工具的具体指令
- **输出格式化** - 指定如何处理工具输出
- **回退指令** - 定义工具失败时该怎么做

### 3. 评估注意事项

- **工具使用指标** - 不仅衡量最终输出质量，还衡量工具使用效果
- **多步骤评估** - 评估基于工具的工作流中的每一步
- **错误率跟踪** - 监控工具故障率和恢复策略
- **上下文保持** - 确保在工具调用之间维护重要上下文

## 限制和注意事项

### 当前限制

- **优化器覆盖范围** - 工具优化在 `FewShotBayesianOptimizer` 或 `ParameterOptimizer` 中不可用
- **工具复杂性** - 非常复杂的工具工作流可能需要手动优化
- **工具可用性** - 优化假设工具在评估期间可用
- **模式更改** - 工具模式修改可能需要重新优化

### 性能注意事项

- **评估成本** - 启用工具的提示词需要更多 LLM 调用进行评估
- **工具延迟** - 外部工具调用会减慢优化速度
- **资源使用** - 复杂的工具工作流可能需要大量计算资源

## 未来路线图

工具优化是一个活跃的开发领域。计划的改进包括：

- **工具特定指标** - 用于工具使用的专门评估指标
- **自动工具发现** - 自动检测和优化工具模式
- **工具性能优化** - 不仅优化提示词，还优化工具使用效率

## 开始使用

要开始优化启用了工具的提示词：

1. **选择支持的优化器** - 除 FewShotBayesianOptimizer 和 ParameterOptimizer 外的任何优化器
2. **定义您的工具** - 创建带有全面描述的清晰工具模式
3. **结构化您的提示词** - 包含工具使用的清晰指令
4. **准备评估数据** - 确保您的数据集包含工具使用场景
5. **运行优化** - 使用标准优化流程处理启用了工具的提示词
