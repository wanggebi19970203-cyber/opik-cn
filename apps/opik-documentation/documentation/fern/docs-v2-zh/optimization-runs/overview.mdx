**Opik Agent Optimizer** 是一款开箱即用的开源智能体和提示词优化 SDK。它利用你已记录到 Opik 的数据集、指标和追踪数据，自动调优提示词、工具和智能体工作流。无需手动编辑指令和重新运行评估，只需选择一个优化器（MetaPrompt、HRPO、Evolutionary、GEPA 等），即可在线或在 Docker 和 Kubernetes 中完全离线地自动迭代优化。

## 为什么团队选择 Opik Agent Optimizer

- **自动提示词优化** - 端到端工作流，几分钟内安装完成，可在本地或你的技术栈中运行。
- **开源且框架无关** - 无供应商锁定，在同一 SDK 中使用 Opik 的第一方优化器或社区热门方案如 GEPA。
- **智能体感知** - 优化范围超越系统提示词，包括 MCP 工具签名、函数调用模式和多步骤智能体工作流。
- **深度可观测性** - 每次试验都将提示词、工具调用、追踪和指标原因记录到 Opik，让你自信地解释和发布变更。

## 工作原理

<Steps>
  <Step title="1. 准备数据和指标">
    使用 Opik 数据集（CSV 上传、API 或追踪导出）加上确定性指标/`ScoreResult` 函数。参见[定义数据集](/v1/agent_optimization/optimization/define_datasets)和[定义指标](/v1/agent_optimization/optimization/define_metrics)。
  </Step>
  <Step title="2. 选择优化器">
    为你的任务选择最佳算法（参见[优化算法](/v1/agent_optimization/algorithms/overview)）。所有优化器暴露相同的 API，因此你可以轻松切换或链式运行。
  </Step>
  <Step title="3. 检查与发布">
    结果会出现在 Opik 仪表板的 **Evaluation → Optimization runs** 中，你可以在推广变更之前比较提示词、失败模式和数据集覆盖范围。
  </Step>
</Steps>

## 优化算法

优化器实现了专有和开源的优化算法。每种算法各有优劣，我们建议首先尝试 GEPA 或 HRPO（分层反思提示词优化器）：

| **算法**                                                                               | **描述**                                                                                                                                                                                                      |
| ------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [MetaPrompt 优化](/v1/agent_optimization/algorithms/metaprompt_optimizer)              | 使用 LLM（"推理模型"）对初始指令提示词进行批判和迭代优化。适用于一般提示词措辞、清晰度和结构改进。**支持 [MCP 工具调用优化](/v1/agent_optimization/algorithms/tool_optimization)。** |
| [HRPO（分层反思提示词优化器）](/v1/agent_optimization/algorithms/hierarchical_adaptive_optimizer) | 使用分层根因分析，通过批量分析失败、综合发现和解决已识别的失败模式来系统性地改进提示词。最适合需要基于理解失败原因进行系统性优化的复杂提示词。 |
| [Few-shot 贝叶斯优化](/v1/agent_optimization/algorithms/fewshot_bayesian_optimizer) | 专为聊天模型设计，使用贝叶斯优化（Optuna）找到最佳数量和组合的 few-shot 示例（演示），作为系统提示词的补充。                      |
| [进化优化](/v1/agent_optimization/algorithms/evolutionary_optimizer)          | 采用遗传算法进化提示词群体。可以发现新颖的提示词结构，支持多目标优化（例如，分数与长度）。可使用 LLM 进行高级变异/交叉。 |
| [GEPA 优化](/v1/agent_optimization/algorithms/gepa_optimizer)                   | 封装外部 GEPA 包，使用反思模型为单轮任务优化单个系统提示词。需要 `pip install gepa`。                                         |
| [参数优化](/v1/agent_optimization/algorithms/parameter_optimizer)                   | 使用贝叶斯优化优化 LLM 调用参数（temperature、top_p 等）。使用 Optuna 进行高效参数搜索，包含全局和局部搜索阶段。最适合在不更改提示词的情况下调优模型行为。                                         |
