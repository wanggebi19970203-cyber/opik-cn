提示词库是管理代理所依赖的所有提示词的中心位置。对它们进行版本控制，从 Opik UI 或 SDK 迭代新版本，并链接产生追踪的精确版本。

## 为什么使用提示词库

当提示词被硬编码时，每次更改都需要修改代码来更新。没有更改历史记录，无法快速回滚，也没有清晰的方式来比较替代版本。

提示词库为驱动代理的提示词提供版本控制。将它们推送到 Opik 一次，之后您可以比较版本、获取特定版本或始终使用最新版本。

## 主要功能

- **项目范围的提示词** — 每个提示词都存在于一个项目中，因此不同代理可以拥有同名提示词而不会冲突。
- **版本控制 `v1`、`v2`、`v3`、…** — 每次更改都会创建一个新的不可变的、顺序编号的版本。使用 `version` 选择器获取特定版本（例如 `"v3"`），或省略它以获取最新版本。
- **文本和聊天提示词** — 对简单的字符串模板以及带有 `{{variable}}` 替换的多消息聊天模板进行版本控制。
- **代理配置集中管理** — 提示词与代理配置的其他部分（模型、参数、工具）放在一起，因此追踪可以链接回产生它的精确配置包。
- **Python 和 TypeScript SDK** — 使用任一语言创建、获取和渲染提示词。

## 工作原理

1. **定义您的提示词** — 使用 `create_prompt` / `create_chat_prompt` 将您的提示词发送到 Opik。这将成为 `v1`。
2. **运行时获取** — 从追踪函数内部调用 `get_prompt` / `get_chat_prompt`。SDK 返回请求的版本并自动将其链接到追踪。
3. **从 UI 迭代** — 编辑提示词并保存新版本。如果您的代码没有固定 `version`，代理将在下次获取时获取新版本 — 否则更新您的代码以指向新版本名称。
