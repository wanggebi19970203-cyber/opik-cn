Opik Agent Optimizer SDK 提供了一套全面的工具，用于优化 LLM 提示词和智能体。本参考指南记录了所有优化器遵循的标准化 API，确保不同优化算法之间的一致性和互操作性。

## 核心特性

- **标准化 API**：所有优化器遵循相同的 `optimize_prompt()` 方法接口
- **多种算法**：支持多种优化策略，包括进化优化、少样本学习、元提示和 GEPA
- **MCP 支持**：内置模型上下文协议（Model Context Protocol）工具调用和优化支持
- **一致的结果**：所有优化器返回标准化的 `OptimizationResult` 对象
- **计数器追踪**：内置 LLM 和工具调用计数器，用于监控使用情况
- **向后兼容**：通过 kwargs 提取保留所有原始参数
- **弃用警告**：对弃用参数提供明确警告和迁移指导

## 核心类

SDK 提供了多个优化器类，它们都继承自 `BaseOptimizer` 并实现相同的标准化接口：

- **ParameterOptimizer**：使用贝叶斯优化来优化 LLM 调用参数（temperature、top_p 等）
- **FewShotBayesianOptimizer**：结合少样本学习与贝叶斯优化
- **MetaPromptOptimizer**：采用元提示技术进行优化
- **EvolutionaryOptimizer**：使用遗传算法进行提示词进化
- **GepaOptimizer**：利用 GEPA（遗传帕累托）优化方法
- **HRPO（分层反思提示优化器）**：使用分层根因分析进行有针对性的提示词精炼

## 标准化方法签名

所有优化器实现以下具有相同签名的核心方法：


以下参数已弃用，将在未来版本中移除：

### 构造函数参数

- 优化器构造函数中的 **`num_threads`**：请改用 `n_threads`

```python
# ❌ 已弃用
optimizer = FewShotBayesianOptimizer(

# ✅ 正确用法
optimizer = FewShotBayesianOptimizer(
```
