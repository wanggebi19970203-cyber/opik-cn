工具优化是一项专门功能，允许您优化使用外部工具和模型上下文协议（MCP）的提示词。除 `FewShotBayesianOptimizer`、`ParameterOptimizer` 和 `GepaOptimizer` 外，所有优化器均支持此功能。

## 什么是工具优化？

工具优化扩展了传统提示词优化，使其能够处理包含以下内容的提示词：

- **MCP 工具** - 用于外部集成的模型上下文协议工具
- **工具模式** - 结构化的工具定义和参数
- **多步骤工作流** - 涉及多个工具的复杂智能体工作流

工具优化**不会**更改工具名称或模式。它会更新：
- 工具**描述**
- 工具**参数描述**

<Note>
  **重要区别：** 许多优化器（包括 GEPA、MetaPrompt 等）可以优化**使用工具的智能体** -
  这意味着优化拥有函数调用或外部工具访问权限的智能体的提示词。然而，**真正的工具优化**
  （优化工具描述和参数描述）是一项独立功能，您可以在支持的优化器中通过 `optimize_tools=True` 启用。
</Note>

<Note>

**工具优化的重要性：**

传统提示词优化侧重于基于文本的提示词，但现代 AI 应用通常需要：

- 与外部 API 和服务集成
- 通过函数调用进行结构化数据处理
- 使用工具进行复杂的多步骤推理
- 基于上下文的动态工具选择

工具优化确保这些复杂的提示词能够像简单的文本提示词一样得到改进。

</Note>

## 支持的工具类型

### 1. 智能体函数调用（非真正的工具优化）

许多优化器可以优化**使用函数调用的智能体**，但这与真正的工具优化不同。以下是 GEPA 优化器的示例：

```python
from opik_optimizer import GepaOptimizer, ChatPrompt

# GEPA 示例：优化使用函数调用的智能体
prompt = ChatPrompt(
    system="You are a helpful assistant. Use the search_wikipedia tool when needed.",
    user="{question}",
    tools=[
        {
            "type": "function",
            "function": {
                "name": "search_wikipedia",
                "description": "This function searches Wikipedia abstracts.",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "query": {
                            "type": "string",
                            "description": "The term or phrase to search for."
                        }
                    },
                    "required": ["query"]
                }
            }
        }
    ],
    function_map={
        "search_wikipedia": lambda query: search_wikipedia(query, use_api=True)
    }
)

# GEPA 优化智能体的提示词，而非工具本身
optimizer = GepaOptimizer(model="gpt-5-nano")
result = optimizer.optimize_prompt(prompt=prompt, dataset=dataset, metric=metric)
```

<Note>
  这是**智能体优化**（优化使用工具的智能体的提示词），而非**工具优化**（优化工具本身）。
</Note>

### 2. MCP（模型上下文协议）工具

**真正的工具优化**适用于支持的优化器中的 MCP 工具和函数调用工具。MCP 工具为外部集成提供标准化接口：

```python
# MCP 工具优化示例
# 完整示例请参见 scripts/litellm_metaprompt_context7_remote_example.py

from opik_optimizer import MetaPromptOptimizer

# MCP 工具配置为 OpenAI 风格的条目（本地或远程）
optimizer = MetaPromptOptimizer(model="gpt-5-nano")

# 任何支持的优化器均可使用（例如 HRPO、Evolutionary、MetaPrompt）。
```

<Note>
MCP 工具优化：

    - 除 FewShotBayesianOptimizer、ParameterOptimizer 和 GepaOptimizer 外，所有优化器均支持
    - 完整示例请参见 `scripts/litellm_metaprompt_context7_remote_example.py`

</Note>

## 工具优化的工作原理

支持的优化器通过专门的优化流程处理启用工具的提示词：

### 1. 工具感知分析

优化器分析：

- **工具模式** - 理解可用函数及其参数
- **工具使用模式** - 提示词中工具的典型调用方式
- **工具依赖关系** - 不同工具之间的关系
- **上下文需求** - 工具正常运行所需的信息

### 2. 提示词-工具集成优化

优化器可以改进：

- **工具选择逻辑** - 更好的工具使用时机指导
- **参数格式化** - 更清晰的工具输入结构指导
- **错误处理** - 处理工具失败或边界情况的指导
- **工具链式调用** - 优化多步骤工具工作流

### 3. 上下文增强

工具优化还改进：

- **输入验证** - 更好的工具输入验证提示词
- **输出处理** - 处理工具输出的指导
- **回退策略** - 工具不可用时的替代方案

## 示例：优化研究助手

让我们看看工具优化如何应用于使用多个工具的研究助手：

```python
from opik_optimizer import MetaPromptOptimizer, ChatPrompt
from opik.evaluation.metrics import LevenshteinRatio

# 定义包含工具的研究助手提示词
research_prompt = ChatPrompt(
    messages=[
        {
            "role": "system",
            "content": """You are a research assistant. When given a research question:
1. Search for relevant information using the search tool
2. Analyze the results using the analysis tool
3. Provide a comprehensive answer based on your findings

Always cite your sources and be thorough in your research."""
        },
        {
            "role": "user",
            "content": "{research_question}"
        }
    ],
    tools=[
        {
            "type": "function",
            "function": {
                "name": "search_academic_database",
                "description": "Search academic papers and research",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "query": {"type": "string"},
                        "year_range": {"type": "string"},
                        "max_results": {"type": "integer"}
                    }
                }
            }
        },
        {
            "type": "function",
            "function": {
                "name": "analyze_text",
                "description": "Analyze and summarize text content",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "text": {"type": "string"},
                        "analysis_type": {"type": "string"}
                    }
                }
            }
        }
    ]
)

# 初始化优化器
optimizer = MetaPromptOptimizer(
    model="openai/gpt-5-nano"
)

# 定义评估指标
def research_quality_metric(dataset_item, llm_output):
    return LevenshteinRatio().score(
        reference=dataset_item['expected_answer'],
        output=llm_output
    )

# 运行优化
result = optimizer.optimize_prompt(
    prompt=research_prompt,
    dataset=research_dataset,
    metric=research_quality_metric,
    n_samples=100,
    max_trials=5
)

print("Optimized prompt with tools:")
print(result.prompt)
```

## 工具优化最佳实践

### 1. 工具模式设计

- **清晰的描述** - 为每个工具提供详细描述
- **完整的参数** - 包含所有必要参数及其类型
- **使用示例** - 在工具描述中有用时添加示例
- **错误处理** - 定义预期的错误条件和响应

### 2. 提示词结构

- **工具介绍** - 向模型清晰说明可用工具
- **使用指南** - 提供工具使用时机和方式的具体指导
- **输出格式化** - 指定工具输出的处理方式
- **回退指令** - 定义工具失败时的处理方式

### 3. 评估注意事项

- **工具使用指标** - 不仅衡量最终输出质量，还要衡量工具使用效率
- **多步骤评估** - 评估基于工具的工作流中的每个步骤
- **错误率跟踪** - 监控工具失败率和恢复策略
- **上下文保持** - 确保在工具调用之间保持重要上下文

## 限制与注意事项

### 当前限制

- **优化器覆盖范围** - `FewShotBayesianOptimizer` 和 `ParameterOptimizer` 不支持工具优化
- **工具复杂度** - 非常复杂的工具工作流可能需要手动优化
- **工具可用性** - 优化假设工具在评估期间可用
- **模式变更** - 工具模式修改可能需要重新优化

### 性能注意事项

- **评估成本** - 启用工具的提示词需要更多 LLM 调用进行评估
- **工具延迟** - 外部工具调用可能减慢优化速度
- **资源使用** - 复杂的工具工作流可能需要大量计算资源

## 未来路线图

工具优化是一个活跃的开发领域。计划的改进包括：

- **工具专用指标** - 针对工具使用的专门评估指标
- **自动工具发现** - 自动检测和优化工具模式
- **工具性能优化** - 不仅优化提示词，还优化工具使用效率

## 开始使用

要开始优化启用工具的提示词：

1. **选择支持的优化器** - 除 FewShotBayesianOptimizer 和 ParameterOptimizer 外的任何优化器
2. **定义工具** - 创建具有完整描述的清晰工具模式
3. **构建提示词** - 包含清晰的工具使用指导
4. **准备评估数据** - 确保数据集包含工具使用场景
5. **运行优化** - 使用标准优化流程处理启用工具的提示词
