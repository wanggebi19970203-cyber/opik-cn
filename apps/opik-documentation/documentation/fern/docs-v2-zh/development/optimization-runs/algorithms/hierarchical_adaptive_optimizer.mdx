`HRPO`（分层反射式提示词优化器）使用分层根因分析来识别和解决提示词中的特定失败模式。它分析评估结果、识别失败模式中的规律，并生成针对性的改进方案以系统性地解决每种失败模式。

<Note>
  在代码中使用 `HRPO` 类（例如 `from opik_optimizer import HRPO`）。底层模块名称仍使用
  `hierarchical_reflective_optimizer` 以保持向后兼容性。
</Note>

<Note>
  `HRPO` 适用于您有一个复杂提示词，希望基于理解其*失败原因*来进行优化的场景。与生成大量随机变体的优化器不同，
  该优化器会系统性地分析失败、识别根因，并进行精准改进以解决每个具体问题。
</Note>

## 工作原理

HRPO（分层反射式提示词优化器）由 Opik 团队开发，用于改进可能已经经过几轮手动提示词工程的提示词。它专注于识别提示词失败的原因，然后更新提示词以解决这些问题。

由于数据集可能很大，我们将分析拆分为多个批次并行处理，然后综合所有批次的分析结果以识别提示词的核心问题。

## 快速开始

您可以使用 `HRPO` 来优化提示词：

```python maxLines=1000
from opik_optimizer import HRPO, ChatPrompt, datasets
from opik.evaluation.metrics.score_result import ScoreResult

# 1. 定义评估数据集
dataset = datasets.hotpot(count=300)  # 或使用您自己的数据集

# 2. 配置评估指标（必须返回原因！）
def answer_quality_metric(dataset_item, llm_output):
    reference = dataset_item.get("answer", "")

    # 您的评分逻辑
    is_correct = reference.lower() in llm_output.lower()
    score = 1.0 if is_correct else 0.0

    # 重要：提供详细的推理说明
    if is_correct:
        reason = f"Output contains the correct answer: '{reference}'"
    else:
        reason = f"Output does not contain expected answer '{reference}'. Output was too vague or incorrect."

    return ScoreResult(
        name="answer_quality",
        value=score,
        reason=reason  # 对根因分析至关重要！
    )

# 3. 定义初始提示词
initial_prompt = ChatPrompt(
    project_name="reflective_optimization",
    messages=[
        {
            "role": "system",
            "content": "You are a helpful assistant that answers questions accurately."
        },
        {
            "role": "user",
            "content": "Question: {question}\n\nProvide a concise answer."
        }
    ]
)

# 4. 初始化 HRPO
optimizer = HRPO(
    model="gpt-4o",
    n_threads=8,
    max_parallel_batches=5,
    seed=42,
    model_parameters={"temperature": 0.7}
)

# 5. 运行优化
optimization_result = optimizer.optimize_prompt(
    prompt=initial_prompt,
    dataset=dataset,
    metric=answer_quality_metric,
    n_samples=100,
    max_trials=5,
    max_retries=2
)

# 6. 查看结果
optimization_result.display()
```

### 模型支持

使用 `HRPO` 时需要考虑两个模型：
- `HRPO.model`：用于根因分析和失败模式综合的模型。
- `ChatPrompt.model`：用于评估提示词的模型。

`model` 参数接受任何 LiteLLM 支持的模型字符串（例如 `"gpt-4o"`、`"azure/gpt-4"`、
`"anthropic/claude-3-opus"`、`"gemini/gemini-1.5-pro"`）。您还可以使用 `model_parameters` 参数传递额外的模型参数：

```python
optimizer = HRPO(
    model="anthropic/claude-3-opus-20240229",
    model_parameters={
        "temperature": 0.7,
        "max_tokens": 4096
    }
)
```
