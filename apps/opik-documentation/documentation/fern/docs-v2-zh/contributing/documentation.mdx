# 为文档做贡献

本指南将帮助你开始为 Opik 文档做贡献。

<Tip>
  在开始之前，请查阅我们的通用[贡献概述](/v1/contributing/overview)和[贡献者许可协议 (CLA)](https://github.com/comet-ml/opik/blob/main/CLA.md)。
</Tip>

## 文档结构

本指南介绍如何为 Opik 文档的两个主要部分做贡献：**本文档网站**：使用 [Fern](https://www.buildwithfern.com/) 构建，以及 **Python SDK 参考文档**：使用 [Sphinx](https://www.sphinx-doc.org/en/master/) 构建。

以下是你如何处理其中任何一个的方式：

<AccordionGroup>
  <Accordion title="为本文档网站做贡献（Fern）">
    本网站（源代码位于 `apps/opik-documentation/documentation`）是我们的主要指南、教程和概念文档所在之处。

    <Steps>
      ### 1. 安装前提条件
      确保你已安装 Node.js 和 npm。你可以按照[此处](https://docs.npmjs.com/downloading-and-installing-node-js-and-npm/)的官方指南操作。

      ### 2. 本地搭建
      ```bash
      cd apps/opik-documentation/documentation

      # 安装依赖 - 只需运行一次
      npm install

      # 可选 - 如果更新 Jupyter Cookbooks 则安装 Python 依赖
      pip install -r requirements.txt

      # 本地运行文档网站
      npm run dev


      ```
      访问本地站点 `http://localhost:3000`。更改将实时更新。

      ### 3. 进行更改
      主要在以下位置更新内容：
      - `fern/docs/`：主要 Markdown 内容（如此页面）。
      - `/docs/cookbook`：我们的 cookbook 和示例集合 - 请注意，你不应直接更新 `cookbook` Markdown 文件，因为它们是从 Jupyter Notebook 生成的。

      请参阅 `docs.yml` 文件了解整体结构和导航。

      ### 4. 提交更改
      对更改满意后，提交它们并向 `comet-ml/opik` 仓库的 `main` 分支发起拉取请求。
    </Steps>

  </Accordion>
  <Accordion title="为 Python SDK 参考文档做贡献（Sphinx）">
    Python SDK 参考文档（源代码位于 `apps/opik-documentation/python-sdk-docs`）是使用 [Sphinx](https://www.sphinx-doc.org/en/master/) 从 Python 代码库中的文档字符串生成的。
    <Steps>
      ### 1. 安装前提条件
      确保你已安装 Python 和 pip。强烈建议使用虚拟环境。

      ### 2. 本地搭建
      ```bash
      cd apps/opik-documentation/python-sdk-docs
      # 安装依赖 - 只需运行一次
      pip install -r requirements.txt
      # 本地运行 Python SDK 参考文档
      make dev
      ```
      访问本地站点 `http://127.0.0.1:8000`。当你修改 SDK（`sdks/python`）中的文档字符串并重建时，更改将实时更新。

      ### 3. 进行更改
      对 SDK 参考的改进通常涉及直接更新位于 `sdks/python` 目录中的 Python 文档字符串。

      ### 4. 构建和预览
      编辑文档字符串后，在 `apps/opik-documentation/python-sdk-docs` 目录中运行 `make dev`（或 `make html` 进行一次性构建）以重新生成 HTML 并预览你的更改。

      ### 5. 提交更改
      将你的更改提交到 Python SDK 源文件和 `python-sdk-docs` 目录中的任何必要更新。向 `main` 分支发起拉取请求。
    </Steps>

  </Accordion>
</AccordionGroup>
