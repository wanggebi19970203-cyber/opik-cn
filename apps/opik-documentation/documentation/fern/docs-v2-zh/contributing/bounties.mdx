# Opik 赏金计划

## 什么是 Opik 赏金计划？

Opik 赏金计划允许像你这样的开发者和贡献者处理特定问题、功能请求或文档改进，以获得金钱奖励。我们相信社区的力量，并希望激励有价值的贡献，帮助 Opik 为每个人变得更好。

## 如何参与

<Steps>
  ### 1. 查阅指南
  在深入之前，请确保你已阅读我们的[贡献者许可协议 (CLA)](https://github.com/comet-ml/opik/blob/main/CLA.md)和主要[贡献指南](/v1/contributing/overview)。这确保你了解我们的开发流程和法律要求。

  ### 2. 寻找赏金任务
  在我们官方的 [Algora 赏金看板](https://algora.io/comet-ml)上浏览开放的赏金任务。寻找与你技能和兴趣匹配的任务。

  ### 3. 了解要求
  每个赏金任务都有清晰的任务描述、预期交付物和奖励金额。在开始工作之前确保你理解需要什么。如果有任何不清楚的地方，请在与赏金相关的 GitHub issue 中请求澄清。

  ### 4. 认领赏金（如适用）
  某些赏金任务可能要求你在 Algora 平台上认领后才能开始工作。请按照赏金列表上提供的具体说明操作。

  ### 5. 进行贡献
  一旦你选定并（如有必要）认领了赏金任务，就可以开始工作了！请遵守我们主要[贡献指南](/v1/contributing/overview)中概述的编码和贡献指南。

  ### 6. 提交贡献
  通常，这涉及向相应的 Opik GitHub 仓库提交拉取请求（PR）。确保在 PR 描述中清晰链接到赏金 issue。

  ### 7. 获得奖励
  一旦你的贡献经过 Opik 团队审查、合并并满足所有赏金要求，你将通过 Algora 平台获得奖励。
</Steps>

## 你可以期待什么样的赏金？

我们将发布各种任务的赏金，包括但不限于：

*   **Bug 修复**：帮助我们识别和修复 Opik 平台（后端、前端、SDK）中的 bug。
*   **功能实现**：贡献新功能和增强。
*   **文档**：改进现有文档、编写新指南或创建教程。
*   **集成**：开发与其他工具和服务的新集成。
*   **测试**：帮助我们提高测试覆盖率并识别问题。

我们期待你的贡献！如果你有任何问题，请随时联系我们。
