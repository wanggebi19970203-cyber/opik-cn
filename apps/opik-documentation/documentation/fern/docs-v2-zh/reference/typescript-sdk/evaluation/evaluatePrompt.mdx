<Note>
  在 Opik 2.0 中，实验是项目范围的。调用 `evaluatePrompt()` 时请确保指定 `projectName`，以便结果与正确的项目关联。
</Note>

`evaluatePrompt` 函数提供了一种简化的方式来评估 prompt 模板对数据集的效果。它自动使用数据集变量格式化消息模板、生成 LLM 响应，并使用指定指标评估结果。

## 概述

`evaluatePrompt` 是 [`evaluate`](/reference/typescript-sdk/evaluation/evaluate_function) 函数的便捷封装，处理：

- **模板格式化** — 自动使用数据集项变量格式化消息模板
- **模型调用** — 使用你指定的模型生成 LLM 响应
- **实验跟踪** — 创建链接到特定 prompt 版本的实验
- **指标评估** — 使用指定指标对输出进行评分

这对于 prompt 工程工作流程特别有用，你想快速测试不同 prompt 模板对数据集的效果。

## 函数签名

```typescript
function evaluatePrompt(
  options: EvaluatePromptOptions
): Promise<EvaluationResult>;
```

### EvaluatePromptOptions

```typescript
interface EvaluatePromptOptions extends Omit<EvaluateOptions, "task"> {
  // 必填参数
  dataset: Dataset;
  messages: OpikMessage[];

  // 可选参数
  model?: SupportedModelId | LanguageModel | OpikBaseModel;
  templateType?: "mustache" | "jinja2";
  scoringMetrics?: BaseMetric[];
  experimentName?: string;
  experimentConfig?: Record<string, unknown>;
  prompts?: Prompt[];
  projectName?: string;
  nbSamples?: number;
  scoringKeyMapping?: Record<string, string>;
}
```

## 参数

### 必填参数

#### dataset

- **类型**：`Dataset`
- **描述**：要评估 prompt 的数据集。每个数据集项将用于格式化消息模板并生成响应。

```typescript
const dataset = await client.getOrCreateDataset("my-dataset", "Evaluation dataset", "my-project");
```

#### messages

- **类型**：`OpikMessage[]`
- **描述**：带有 `{{placeholders}}` 的消息模板数组，将使用数据集变量进行格式化。

```typescript
messages: [
  { role: "system", content: "You are a helpful assistant" },
  { role: "user", content: "Translate to {{language}}: {{text}}" },
];
```

### 可选参数

#### model

- **类型**：`SupportedModelId | LanguageModel | OpikBaseModel`
- **默认值**：`"gpt-5-nano"`
- **描述**：用于生成的语言模型。可以是：
  - 模型 ID 字符串（例如 `"gpt-5-nano"`、`"claude-3-5-sonnet-latest"`、`"gemini-2.0-flash"`）
  - Vercel AI SDK 中预配置的 `LanguageModel` 实例
  - 自定义 `OpikBaseModel` 实现

```typescript
// 使用模型 ID 字符串
model: "gpt-5-nano";

// 使用 LanguageModel 实例
import { openai } from "@ai-sdk/openai";
const customModel = openai("gpt-5-nano");
model: customModel;
```

#### templateType

- **类型**：`"mustache" | "jinja2"`
- **默认值**：`"mustache"`
- **描述**：用于消息内容中变量替换的模板引擎。

```typescript
// Mustache 语法（默认）
templateType: "mustache";
messages: [{ role: "user", content: "Hello {{name}}" }];

// Jinja2 语法
templateType: "jinja2";
messages: [{ role: "user", content: "Hello {{ name }}" }];
```

#### scoringMetrics

- **类型**：`BaseMetric[]`
- **描述**：用于评估生成输出的指标数组。可以包括启发式和 LLM Judge 指标。

```typescript
import { ExactMatch, Hallucination } from "opik";

scoringMetrics: [new ExactMatch(), new Hallucination()];
```

#### experimentName

- **类型**：`string`
- **描述**：实验名称。如果未提供，将自动生成名称。

```typescript
experimentName: "Prompt Evaluation - Translation Task";
```

#### experimentConfig

- **类型**：`Record<string, unknown>`
- **描述**：与实验一起存储的附加元数据。该函数自动将 `prompt_template` 和 `model` 添加到此配置中。

```typescript
experimentConfig: {
  temperature: 0.7,
  max_tokens: 1000,
  version: "v2",
};
```

#### prompts

- **类型**：`Prompt[]`
- **描述**：要链接到此实验的 Opik Prompt 对象数组。用于跟踪使用了哪些 prompt 版本。

```typescript
const prompt = new Prompt({
  name: "translation-prompt",
  prompt: "Translate to {{language}}: {{text}}",
  projectName: "my-project",
});

prompts: [prompt];
```

#### projectName

- **类型**：`string`
- **描述**：要记录 trace 的 Opik 项目名称。

```typescript
projectName: "prompt-engineering";
```

#### nbSamples

- **类型**：`number`
- **描述**：要评估的最大数据集项数。适用于快速测试。

```typescript
nbSamples: 10; // 仅评估前 10 项
```

#### scoringKeyMapping

- **类型**：`Record<string, string>`
- **描述**：当指标参数名称与数据集/输出字段名称不匹配时进行映射。

```typescript
scoringKeyMapping: {
  input: "question", // 将 'input' 参数映射到 'question' 字段
  expected: "reference_answer", // 将 'expected' 参数映射到 'reference_answer' 字段
};
```

## 返回值

返回一个 `Promise<EvaluationResult>`，包含：

```typescript
interface EvaluationResult {
  experimentId: string; // 创建的实验 ID
  experimentName: string; // 实验名称
  testResults: TestResult[]; // 每个数据集项的结果
}
```

## 示例

### 基本用法

使用默认设置的简单 prompt 评估：

```typescript
import { Opik, evaluatePrompt } from "opik";

const client = new Opik();
const dataset = await client.getOrCreateDataset("qa-dataset", "Evaluation dataset", "my-project");

await dataset.insert([
  {
    question: "What is the capital of France?",
    expected_answer: "Paris",
  },
  {
    question: "How do you calculate the area of a circle?",
    expected_answer: "π × radius²",
  },
]);

const result = await evaluatePrompt({
  dataset,
  messages: [
    {
      role: "system",
      content:
        "You are a helpful assistant. Answer questions accurately and concisely.",
    },
    { role: "user", content: "{{question}}" },
  ],
  model: "gpt-5-nano",
  projectName: "my-project",
});

console.log(`Experiment ID: ${result.experimentId}`);
console.log(`Evaluated ${result.testResults.length} items`);
```

### 使用评分指标

使用自动评分评估 prompt：

```typescript
import { evaluatePrompt } from "opik";
import { Hallucination, ExactMatch } from "opik";

// 创建带预期答案的数据集
const dataset = await client.getOrCreateDataset("geography-qa", "Geography evaluation dataset", "my-project");
await dataset.insert([
  {
    country: "France",
    expected_answer: "Paris",
  },
  {
    country: "Japan",
    expected_answer: "Tokyo",
  },
]);

await evaluatePrompt({
  dataset,
  messages: [
    {
      role: "user",
      content: "What is the capital of {{country}}?",
    },
  ],
  model: "gpt-5-nano",
  scoringMetrics: [
    new ExactMatch(), // 检查与预期输出的精确匹配
    new Hallucination(), // 检查幻觉
  ],
  experimentName: "Geography Quiz Evaluation",
  projectName: "my-project",
});
```

### 使用 LanguageModel 实例

使用 LanguageModel 实例获取提供商特定功能：

```typescript
import { openai } from "@ai-sdk/openai";
import { evaluatePrompt } from "opik";

// 创建模型实例
const customModel = openai("gpt-5-nano");

await evaluatePrompt({
  dataset,
  messages: [{ role: "user", content: "Summarize: {{text}}" }],
  model: customModel,
  experimentConfig: {
    model_provider: "openai",
    model_name: "gpt-5-nano",
  },
  projectName: "my-project",
});
```

### 多提供商模型支持

该函数支持来自多个提供商的模型：

```typescript
// OpenAI
model: "gpt-5-nano";

// Anthropic
model: "claude-3-5-sonnet-latest";

// Google Gemini
model: "gemini-2.0-flash";

// 或使用提供商特定的 LanguageModel 实例
import { anthropic } from "@ai-sdk/anthropic";
const claude = anthropic("claude-3-5-sonnet-latest");
model: claude;
```

### 链接到 Prompt 版本

跟踪评估中使用了哪些 prompt 版本：

```typescript
import { Opik, Prompt, evaluatePrompt } from "opik";

// 创建或获取 prompt
const prompt = new Prompt({
  name: "customer-support-prompt",
  prompt: "{{system_message}}\n\nUser: {{user_query}}",
  projectName: "my-project",
});

// 将 prompt 链接到评估
await evaluatePrompt({
  dataset,
  messages: [
    { role: "system", content: "{{system_message}}" },
    { role: "user", content: "{{user_query}}" },
  ],
  model: "gpt-5-nano",
  prompts: [prompt], // 链接到 prompt
  experimentName: "Customer Support - v2.1",
  projectName: "my-project",
});
```

### 模板类型

#### Mustache 模板（默认）

```typescript
await evaluatePrompt({
  dataset,
  messages: [
    {
      role: "user",
      content: "Hello {{name}}, your order #{{order_id}} is ready.",
    },
  ],
  templateType: "mustache", // 这是默认值
  projectName: "my-project",
});
```

#### Jinja2 模板

```typescript
await evaluatePrompt({
  dataset,
  messages: [
    {
      role: "user",
      content: "Hello {{ name }}, your order #{{ order_id }} is ready.",
    },
  ],
  templateType: "jinja2",
  projectName: "my-project",
});
```

### 评分键映射

将数据集字段映射到指标参数名称：

```typescript
// 数据集有：{ question: "...", reference_answer: "..." }
// 指标期望：{ input: "...", expected: "..." }

await evaluatePrompt({
  dataset,
  messages: [{ role: "user", content: "{{question}}" }],
  scoringMetrics: [new ExactMatch()],
  scoringKeyMapping: {
    input: "question",
    expected: "reference_answer",
  },
  projectName: "my-project",
});
```

### 子集评估

仅评估数据集的子集以快速迭代：

```typescript
await evaluatePrompt({
  dataset,
  messages: [{ role: "user", content: "{{prompt}}" }],
  nbSamples: 5, // 仅评估前 5 项
  experimentName: "Quick Test",
  projectName: "my-project",
});
```

## 工作原理

调用 `evaluatePrompt` 时，会发生以下情况：

1. **模板格式化**：对于每个数据集项，消息模板使用项变量进行格式化
2. **模型调用**：格式化后的消息发送到指定模型以生成响应
3. **实验创建**：创建（或更新）带有元数据的实验
4. **指标评分**：如果提供了指标，每个输出都会被评分
5. **结果聚合**：结果被收集并返回

## 实验配置

该函数自动使用以下内容丰富实验配置：

- `prompt_template`：使用的消息模板
- `model`：模型标识符（名称或类型）

你可以通过 `experimentConfig` 添加额外的元数据：

```typescript
experimentConfig: {
  // evaluatePrompt 自动添加：
  // prompt_template: [{ role: 'user', content: '...' }]
  // model: 'gpt-5-nano'

  // 你的自定义元数据：
  temperature: 0.7,
  version: "v2.0",
  author: "team-ai",
  description: "Testing improved prompt structure",
};
```

## 最佳实践

### 1. 从简单开始，然后添加指标

从基本 prompt 评估开始，然后根据需要添加指标：

```typescript
// 步骤 1：基本评估以查看输出
await evaluatePrompt({
  dataset,
  messages: [{ role: "user", content: "{{input}}" }],
  projectName: "my-project",
});

// 步骤 2：审查输出后添加指标
await evaluatePrompt({
  dataset,
  messages: [{ role: "user", content: "{{input}}" }],
  scoringMetrics: [new Hallucination()],
  projectName: "my-project",
});
```

### 2. 使用描述性实验名称

便于查找和比较实验：

```typescript
experimentName: "Translation - GPT-4o - v2.3 - 2025-01-15";
```

### 3. 对 Prompt 进行版本控制

将评估链接到 prompt 版本以获得更好的跟踪：

```typescript
const prompt = await client.createPrompt({
  name: "qa-prompt",
  prompt: "Answer: {{question}}",
  version: "v2.3",
});

await evaluatePrompt({
  dataset,
  messages: [{ role: "user", content: "Answer: {{question}}" }],
  prompts: [prompt],
  projectName: "my-project",
});
```

### 4. 从小样本开始

使用 `nbSamples` 在完整评估前快速迭代：

```typescript
// 使用 10 个样本快速测试
await evaluatePrompt({
  dataset,
  messages: [{ role: "user", content: "{{input}}" }],
  nbSamples: 10,
  projectName: "my-project",
});

// 满意后进行完整评估
await evaluatePrompt({
  dataset,
  messages: [{ role: "user", content: "{{input}}" }],
  projectName: "my-project",
  // 评估整个数据集
});
```

### 5. 在系统消息中包含上下文

使用清晰的系统消息构建 prompt：

```typescript
messages: [
  {
    role: "system",
    content:
      "You are an expert {{domain}} assistant. Provide accurate, concise answers.",
  },
  {
    role: "user",
    content: "{{question}}",
  },
];
```

## 错误处理

该函数验证输入并对常见问题抛出错误：

```typescript
try {
  await evaluatePrompt({
    dataset,
    messages: [],
  });
} catch (error) {
  console.error(error.message);
  // 错误：Messages 数组是必需的且不能为空
}
```

常见验证错误：

- 缺少必填的 `dataset` 参数
- 空的 `messages` 数组
- 无效的 `experimentConfig`（必须是普通对象）
- 无效的 `templateType`（必须是 'mustache' 或 'jinja2'）
