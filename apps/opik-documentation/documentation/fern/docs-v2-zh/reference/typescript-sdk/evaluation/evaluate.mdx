<Note>
  在 Opik 2.0 中，实验是项目范围的。调用 `evaluate()` 时请确保指定 `projectName`，以便结果与正确的项目关联。
</Note>

`evaluate` 函数允许你使用可自定义指标对数据集运行 LLM 任务的综合评估。

```typescript
async function evaluate(options: EvaluateOptions): Promise<EvaluationResult>;
```

## 参数

该函数接受一个类型为 `EvaluateOptions` 的 `options` 参数，包含以下属性：

| 参数           | 类型                      | 必填 | 描述                                                                                |
| ------------------- | ------------------------- | -------- | ------------------------------------------------------------------------------------------ |
| `dataset`           | `Dataset \| DatasetVersion` | 是    | 要评估的数据集或数据集版本。使用 `DatasetVersion` 进行固定到特定快照的可重现评估。 |
| `task`              | `EvaluationTask`          | 是      | 要执行的特定 LLM 任务                                                           |
| `scoringMetrics`    | `BaseMetric[]`            | 否       | 可选的指标数组，用于评估模型性能（例如准确率、F1 分数）         |
| `experimentName`    | `string`                  | 否       | 可选的评估实验名称，用于跟踪和报告                    |
| `projectName`       | `string`                  | 否       | 可选的项目标识符，用于关联此实验                              |
| `experimentConfig`  | `Record<string, unknown>` | 否       | 可选的实验配置设置，以键值对形式                      |
| `nbSamples`         | `number`                  | 否       | 可选的要评估的数据集样本数量（如未指定则默认为全部） |
| `client`            | `OpikClient`              | 否       | 可选的 Opik 客户端实例，用于跟踪                                          |
| `scoringKeyMapping` | `ScoringKeyMappingType`   | 否       | 可选的数据集键和评分指标输入之间的映射                            |

## 返回值

该函数返回一个 Promise，解析为包含以下内容的 `EvaluationResult` 对象：

- 所有评估样本的聚合分数
- 单个样本结果
- 执行元数据

## 使用示例

```typescript
import {
  evaluate,
  EvaluationTask,
  Opik,
  BaseMetric,
  EvaluationScoreResult,
  ExactMatch,
} from "opik";
import OpenAI from "openai";

// 初始化客户端
const openai = new OpenAI();
const opik = new Opik();

// 定义数据集项类型
type DatasetItem = {
  input: string;
  expected_output: string;
  metadata: {
    category: string;
    difficulty: string;
    version: number;
  };
};

// 定义 LLM 任务
const llmTask: EvaluationTask<DatasetItem> = async (datasetItem) => {
  const { input } = datasetItem;

  const response = await openai.responses.create({
    model: "gpt-5-nano",
    instructions: "You are a coding assistant",
    input,
  });

  return { output: response.output_text };
};

async function runEvaluation() {
  // 获取或创建数据集
  const dataset = await opik.getOrCreateDataset<DatasetItem>("example-dataset", "Evaluation dataset", "my-project");

  // 运行评估
  const result = await evaluate({
    dataset,
    task: llmTask,
    scoringMetrics: [new ExactMatch()],
    experimentName: "Example Evaluation",
    projectName: "my-project",

    // 将任务输出和数据集项数据映射到预期的指标输入
    scoringKeyMapping: {
      expected: "expected_output",
    },
  });
}
```

### 评估特定版本

对于可重现的评估，使用 `DatasetVersion` 而不是 `Dataset`：

```typescript
// 获取特定版本用于可重现评估
const dataset = await opik.getOrCreateDataset<DatasetItem>("example-dataset", "Evaluation dataset", "my-project");
const v2 = await dataset.getVersionView("v2");

const result = await evaluate({
  dataset: v2,
  task: llmTask,
  scoringMetrics: [new ExactMatch()],
  experimentName: "Pinned to v2",
  projectName: "my-project",
});
// 实验链接到版本 v2，而非最新版本
```

## 备注

- 该函数自动在 Opik 中创建实验用于跟踪和分析
- 如果未提供 `client`，则使用全局 Opik 客户端实例
- 你可以提供类型参数来正确类型化你的数据集和任务输入/输出
- 评估过程中的错误将被正确记录并重新抛出
