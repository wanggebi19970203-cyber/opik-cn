REST API 可同时用于开源平台和 Opik Cloud。两者的主要区别在于使用的 URL 和用于身份验证的请求头。

<Tabs>
    <Tab title="开源平台">
    如果你使用的是开源平台，可以直接使用 `http://localhost:5173/api/v1/<endpoint_path>`：

    ```bash
    curl -X GET 'http://localhost:5173/api/v1/private/projects'
    ```

    </Tab>
    <Tab title="Opik Cloud">
    如果你使用的是 Opik Cloud 或 Comet 平台的私有化部署版本，需要在请求头中指定 Opik
    API Key 和 Opik Workspace：

    ```bash
    curl -X GET 'https://www.comet.com/opik/api/v1/private/projects' \
    -H 'Accept: application/json' \
    -H 'Comet-Workspace: <your-workspace-name>' \
    -H 'authorization: <your-api-key>'
    ```

    <Warning>
    请注意，authorization 请求头的值不包含 `Bearer ` 前缀。
    </Warning>
    </Tab>

</Tabs>
