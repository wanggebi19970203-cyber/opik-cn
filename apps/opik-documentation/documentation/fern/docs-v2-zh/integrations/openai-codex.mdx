[OpenAI Codex](https://developers.openai.com/codex) 支持通过 Codex 配置文件选择性启用 OpenTelemetry 导出。

## 适用场景

如果您使用 Codex（CLI/IDE/应用）并希望其 OTEL 追踪导出器将遥测数据发送到 Opik，请使用本指南。

<Callout type="info">
以下代码块结构遵循当前 Codex 运行时配置格式，用于本地 `config.toml` 文件（`[otel.trace_exporter.otlp-http]`）。
</Callout>

## Codex 配置位置

Codex 从以下位置读取配置：
- 用户配置：`~/.codex/config.toml`
- 项目配置：`.codex/config.toml`

参见 [Codex 配置基础](https://developers.openai.com/codex/config-basic)。

## Opik OTLP 追踪端点模式

有关 Opik OTEL 端点行为，请参阅 [Opik OpenTelemetry 概述](/integrations/opentelemetry)。

<Tabs>
    <Tab value="Opik Cloud" title="Opik Cloud">
        ```toml
        [otel]
        # Codex 遥测中的可选环境标签
        trace_exporter = "otlp-http"
        environment = "prod"
        log_user_prompt = false

        [otel.trace_exporter.otlp-http]
        endpoint = "https://www.comet.com/opik/api/v1/private/otel/v1/traces"
        protocol = "binary"
        headers = { "Authorization" = "<your-api-key>", "Comet-Workspace" = "<your-workspace>", "projectName" = "<your-project-name>" }
        ```

        必需的请求头：
        - `Authorization`
        - `Comet-Workspace`

        可选的请求头：
        - `projectName`（推荐）
    </Tab>
    <Tab value="Enterprise deployment" title="企业部署">
        ```toml
        [otel]
        trace_exporter = "otlp-http"
        environment = "prod"
        log_user_prompt = false

        [otel.trace_exporter.otlp-http]
        endpoint = "https://<comet-deployment-url>/opik/api/v1/private/otel/v1/traces"
        protocol = "binary"
        headers = { "Authorization" = "<your-api-key>", "Comet-Workspace" = "<your-workspace>", "projectName" = "<your-project-name>" }
        ```

        必需的请求头：
        - `Authorization`
        - `Comet-Workspace`

        可选的请求头：
        - `projectName`（推荐）
    </Tab>
    <Tab value="Self-hosted instance" title="自托管实例">
        ```toml
        [otel]
        trace_exporter = "otlp-http"
        environment = "prod"
        log_user_prompt = false

        [otel.trace_exporter.otlp-http]
        endpoint = "http://localhost:5173/api/v1/private/otel/v1/traces"
        protocol = "binary"
        headers = { "projectName" = "<your-project-name>" }
        ```

        必需的请求头：
        - 默认无（取决于您的自托管认证配置）

        可选的请求头：
        - `projectName`（推荐）
        - 如果您的实例启用了认证，则需要认证请求头
    </Tab>
</Tabs>

## 示例意图和最小有效配置

意图：
将 Codex OTEL 追踪导出路由到 Opik，并附加项目/工作区归属信息。

适用条件：
您已在配置中启用 Codex OTEL 导出并选择 OTLP/HTTP 导出器。

必需字段：
- `[otel]` 下的 `trace_exporter = "otlp-http"`
- `otel.trace_exporter` 导出器配置块（`otlp-http`）
- `endpoint`
- `protocol`（`binary` 或 `json`，推荐 binary）

可选字段：
- `headers`（强烈推荐设置 `projectName`）
- `otel.environment`
- `otel.log_user_prompt`（除非策略允许导出提示词，否则保持 `false`）

最小有效配置：

```toml
[otel]
trace_exporter = "otlp-http"
log_user_prompt = false

[otel.trace_exporter.otlp-http]
endpoint = "https://www.comet.com/opik/api/v1/private/otel/v1/traces"
protocol = "binary"
headers = { "Authorization" = "<your-api-key>", "Comet-Workspace" = "<your-workspace>", "projectName" = "<your-project-name>" }
```

## 验证

1. 更新 `config.toml` 后运行一个 Codex 会话。
2. 确认 OTLP HTTP 请求已发送到 `/otel/v1/traces`。
3. 验证追踪数据出现在预期的 Opik 工作区/项目中。

## 注意事项

- Codex 遥测导出是选择性启用的。
- 除非策略明确允许导出提示词文本，否则请保持 `log_user_prompt = false`。
- 如果您的 Codex 版本使用不同的导出器键路径，请根据您安装版本的配置参考进行调整。
