[AWS Bedrock](https://aws.amazon.com/bedrock/) 是一项全托管服务，通过单一 API 提供来自 AI21 Labs、Anthropic、Cohere、Meta、Mistral AI、Stability AI 和 Amazon 等领先 AI 公司的高性能基础模型 (FM) 访问。

本指南介绍如何将 Opik 与 Bedrock Python SDK 集成，支持 **Converse API** 和 **Invoke Model API**。通过使用 Opik 提供的 `track_bedrock` 方法，您可以轻松地在 Opik 项目中跟踪和评估 Bedrock API 调用，Opik 将自动记录输入提示词、使用的模型、token 用量和生成的响应。

> 您也可以在本地运行 Opik 平台，请参阅[安装指南](https://www.comet.com/docs/opik/self-host/overview/?from=llm&utm_source=opik&utm_medium=colab&utm_content=bedrock&utm_campaign=opik) 了解更多信息。

## 快速开始

### 安装

要开始跟踪您的 Bedrock LLM 调用，您需要安装 `opik` 和 `boto3` 包。可以使用 pip 安装：

```bash
pip install opik boto3
```

### 配置 Opik

根据您的部署类型配置 Opik Python SDK。详细说明请参阅 [Python SDK 配置指南](/tracing/advanced/sdk_configuration)，了解以下内容：

- **CLI 配置**：`opik configure`
- **代码配置**：`opik.configure()`
- **自托管 vs 云托管 vs 企业版**设置
- **配置文件**和环境变量

### 配置 Bedrock

要配置 Bedrock，您需要：

- 为 boto 配置 AWS 凭证，请参阅[以下文档页面了解如何设置](https://boto3.amazonaws.com/v1/documentation/api/latest/guide/credentials.html)。
- 拥有您计划使用的模型的访问权限，请参阅[以下文档页面了解如何操作](https://docs.aws.amazon.com/bedrock/latest/userguide/model-access-modify.html)。

您可以在 [AWS Bedrock 控制台](https://us-east-1.console.aws.amazon.com/bedrock/home?region=us-east-1#/providers?model=meta.llama3-2-3b-instruct-v1:0) 中请求模型访问权限。

完成上述步骤后，您可以创建 boto3 客户端：

```python
import boto3

REGION = "us-east-1"
MODEL_ID = "us.meta.llama3-2-3b-instruct-v1:0"

bedrock_client = boto3.client(
    service_name="bedrock-runtime",
    region_name=REGION,
    # aws_access_key_id=ACCESS_KEY,
    # aws_secret_access_key=SECRET_KEY,
    # aws_session_token=SESSION_TOKEN,
)
```

## 记录 LLM 调用

Opik 支持两种 AWS Bedrock API：**Converse API**（统一接口）和 **Invoke Model API**（模型特定格式）。要将 LLM 调用记录到 Opik，请使用 `track_bedrock` 包装您的 boto3 客户端：

```python
import os
from opik.integrations.bedrock import track_bedrock

# 通过环境变量设置项目名称
os.environ["OPIK_PROJECT_NAME"] = "bedrock-integration-demo"

bedrock_client = track_bedrock(bedrock_client)
```

<Tip>
  尽管 Invoke Model API 对每个模型提供商使用不同的输入/输出格式，Opik 会自动处理所有支持模型的格式检测和成本跟踪，为不同模型格式提供统一的可观测性。
</Tip>

### Converse API（统一接口）

Converse API 为所有支持的模型提供统一接口：

```python
import os
import boto3
from opik.integrations.bedrock import track_bedrock

# 通过环境变量设置项目名称
os.environ["OPIK_PROJECT_NAME"] = "bedrock-integration-demo"

# 初始化并跟踪 Bedrock 客户端
bedrock_client = boto3.client("bedrock-runtime", region_name="us-east-1")
bedrock_client = track_bedrock(bedrock_client)

PROMPT = "Why is it important to use a LLM Monitoring like CometML Opik tool that allows you to log traces and spans when working with LLM Models hosted on AWS Bedrock?"

response = bedrock_client.converse(
    modelId="us.meta.llama3-2-3b-instruct-v1:0",
    messages=[{"role": "user", "content": [{"text": PROMPT}]}],
    inferenceConfig={"temperature": 0.5, "maxTokens": 512, "topP": 0.9},
)
print("Response", response["output"]["message"]["content"][0]["text"])
```

## 流式 API

两种 Bedrock API 都支持流式响应，适用于实时应用场景。Opik 会自动跟踪两种 API 的流式调用。

### Converse Stream API

`converse_stream` 方法提供统一接口的流式传输：

```python
import os
import boto3
from opik.integrations.bedrock import track_bedrock

# 通过环境变量设置项目名称
os.environ["OPIK_PROJECT_NAME"] = "bedrock-integration-demo"

# 初始化并跟踪 Bedrock 客户端
bedrock_client = boto3.client("bedrock-runtime", region_name="us-east-1")
bedrock_client = track_bedrock(bedrock_client)

def stream_conversation(
    bedrock_client,
    model_id,
    messages,
    system_prompts,
    inference_config,
):
    """
    向模型发送消息并使用 Converse API 流式传输响应。
    Args:
        bedrock_client: Boto3 Bedrock 运行时客户端。
        model_id (str): 要使用的模型 ID。
        messages (JSON): 要发送的消息。
        system_prompts (JSON): 要发送的系统提示词。
        inference_config (JSON): 要使用的推理配置。

    Returns:
        无。
    """
    response = bedrock_client.converse_stream(
        modelId=model_id,
        messages=messages,
        system=system_prompts,
        inferenceConfig=inference_config,
    )

    stream = response.get("stream")
    if stream:
        for event in stream:
            if "messageStart" in event:
                print(f"\nRole: {event['messageStart']['role']}")

            if "contentBlockDelta" in event:
                print(event["contentBlockDelta"]["delta"]["text"], end="")

            if "messageStop" in event:
                print(f"\nStop reason: {event['messageStop']['stopReason']}")

            if "metadata" in event:
                metadata = event["metadata"]
                if "usage" in metadata:
                    print("\nToken usage")
                    print(f"Input tokens: {metadata['usage']['inputTokens']}")
                    print(f"Output tokens: {metadata['usage']['outputTokens']}")
                    print(f"Total tokens: {metadata['usage']['totalTokens']}")

# 示例用法
system_prompt = """You are an app that creates playlists for a radio station
  that plays rock and pop music. Only return song names and the artist."""

input_text = "Create a list of 3 pop songs."
messages = [{"role": "user", "content": [{"text": input_text}]}]
system_prompts = [{"text": system_prompt}]
inference_config = {"temperature": 0.5, "topP": 0.9}

stream_conversation(
    bedrock_client,
    "us.meta.llama3-2-3b-instruct-v1:0",
    messages,
    system_prompts,
    inference_config,
)
```

## 高级用法

### 与 `@track` 装饰器配合使用

如果您的 LLM 管道中有多个步骤，可以使用 `@track` 装饰器记录每个步骤的 trace。如果在某个步骤中调用了 Bedrock，该 LLM 调用将与对应的步骤关联：

```python
import boto3
from opik import track
from opik.integrations.bedrock import track_bedrock

# 初始化并跟踪 Bedrock 客户端
bedrock_client = boto3.client("bedrock-runtime", region_name="us-east-1")
bedrock_client = track_bedrock(bedrock_client, project_name="bedrock-integration-demo")

MODEL_ID = "us.anthropic.claude-3-5-sonnet-20241022-v2:0"

@track
def generate_story(prompt):
    res = bedrock_client.converse(
        modelId=MODEL_ID, 
        messages=[{"role": "user", "content": [{"text": prompt}]}],
        inferenceConfig={"temperature": 0.7, "maxTokens": 1000}
    )
    return res["output"]["message"]["content"][0]["text"]

@track
def generate_topic():
    prompt = "Generate a topic for a story about Opik."
    res = bedrock_client.converse(
        modelId=MODEL_ID, 
        messages=[{"role": "user", "content": [{"text": prompt}]}],
        inferenceConfig={"temperature": 0.7, "maxTokens": 500}
    )
    return res["output"]["message"]["content"][0]["text"]

@track
def generate_opik_story():
    topic = generate_topic()
    story = generate_story(topic)
    return story

# 执行多步骤管道
generate_opik_story()
```

## 成本跟踪

`track_bedrock` 包装器会自动跟踪所有支持的 AWS Bedrock 模型的 token 用量和成本，无论您使用的是 Converse API 还是 Invoke Model API。

<Tip>
  尽管通过 InvokeModel API 访问的模型（Anthropic、Amazon、Meta、Mistral）使用不同的输入/输出格式，Opik 会自动检测响应格式并为所有模型提取统一的成本和用量信息。因此，即使您无法使用统一的 Converse API，通过我们的集成仍然可以获得主要的跟踪收益。
</Tip>

成本信息会自动捕获并显示在 Opik UI 中，包括：

- Token 用量详情
- 基于 Bedrock 定价的每次请求成本
- 总 trace 成本

<Tip>
  在[支持的模型](/tracing/advanced/cost_tracking)页面查看完整的支持模型和提供商列表。
</Tip>
