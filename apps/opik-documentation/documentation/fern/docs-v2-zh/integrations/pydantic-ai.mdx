[Pydantic AI](https://ai.pydantic.dev/) 是一个 Python 代理框架，专为使用生成式 AI 构建生产级应用而设计。

Pydantic AI 的主要优势在于集成了 Pydantic 的类型安全数据验证，确保 AI 应用中的响应结构化且可靠。

## 快速入门

### 安装

要使用 Pydantic AI 与 Opik 的集成，您需要安装 Pydantic AI 和 logfire：

```bash
pip install --upgrade pydantic-ai logfire 'logfire[httpx]'
```

### 配置 Pydantic AI

使用 Pydantic AI 之前，您需要配置 LLM 提供商的 API 密钥。本示例使用 OpenAI，您可以在[这些页面中查找或创建 API 密钥](https://platform.openai.com/settings/organization/api-keys)：

您可以通过环境变量设置：

```bash
export OPENAI_API_KEY="YOUR_API_KEY"
```

或通过编程方式设置：

```python
import os
import getpass

if "OPENAI_API_KEY" not in os.environ:
    os.environ["OPENAI_API_KEY"] = getpass.getpass("Enter your OpenAI API key: ")
```

## 在 Pydantic AI 中使用 Opik

要追踪您的 Pydantic AI 代理，需要配置 logfire，因为 Pydantic AI 使用该框架来启用追踪功能。

```python
import logfire

logfire.configure(
    send_to_logfire=False,
)
logfire.instrument_pydantic_ai()
```

## 实际示例

完成所有配置后，您可以创建并运行 Pydantic AI 代理：

```python
import nest_asyncio
from pydantic_ai import Agent

# 在 Jupyter notebooks 中启用异步支持
nest_asyncio.apply()

# 创建一个简单的代理
agent = Agent(
    "openai:gpt-4o",
    system_prompt="Be concise, reply with one sentence.",
)

# 运行代理
result = agent.run_sync('Where does "hello world" come from?')
print(result.output)
```

## 记录会话线程

您可以通过在根 Logfire span 上设置 `thread_id` 作为 span 属性，将多个代理调用分组到一个会话线程中。Opik 的 OTEL 摄取功能会识别此属性并将其直接映射到追踪的 `thread_id` 字段：

```python
# Logfire 封装了 OTEL - thread_id 自动成为 span 属性
with logfire.span("chat_turn", thread_id=thread_id):
    result = agent.run_sync("What is machine learning?")
```

## 与 `@track` 结合使用

如果将代理调用包装在 `@track` 装饰的函数中（例如，为了捕获带有独立输入/输出的清晰入口点），Pydantic AI / logfire 的 span 和 `@track` span 通常会分布在两个独立的追踪中，因为 logfire 生成 OpenTelemetry span，而 `@track` 保持自己的上下文。

在 logfire 的 tracer provider 上注册 `OpikSpanProcessor`，可以将它们合并到单个追踪中。该处理器会自动将 OpenTelemetry span 链接到活动的 `@track` span，无需头部传播或每次调用的额外配置：

```python
import opik
import logfire
from pydantic_ai import Agent
from opik.integrations.otel import OpikSpanProcessor

logfire.configure(
    send_to_logfire=False,
    additional_span_processors=[OpikSpanProcessor()],
)
logfire.instrument_pydantic_ai()

agent = Agent("openai:gpt-4o")

@opik.track
def run(question: str) -> str:
    return agent.run_sync(question).output

run('Where does "hello world" come from?')
```

结果是一个以 `run` 入口点为根、`agent run` / 模型 span 嵌套在下方的单一追踪。有关通用机制，请参阅[将 OpenTelemetry span 链接到现有 Opik 追踪](/integrations/opentelemetry-python-sdk#linking-opentelemetry-spans-to-an-existing-opik-trace)。
