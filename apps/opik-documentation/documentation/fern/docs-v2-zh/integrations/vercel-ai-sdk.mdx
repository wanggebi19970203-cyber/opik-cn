## 设置

AI SDK 通过 OpenTelemetry 支持追踪功能。借助 `OpikExporter`，你可以在 Opik 中采集这些追踪数据。
在遥测功能处于实验阶段期间（[文档](https://sdk.vercel.ai/docs/ai-sdk-core/telemetry#enabling-telemetry)），你可以通过在每个需要追踪的请求上设置 `experimental_telemetry` 来启用它。

```ts
const result = await generateText({
  model: openai("gpt-4o"),
  prompt: "Tell a joke",
  experimental_telemetry: { isEnabled: true },
});
```

要在 Opik 中采集追踪数据，你需要将 `OpikExporter` 添加到应用中。首先，设置环境变量：

意图：
使用 `OpikExporter` 作为 AI SDK 遥测的 OTEL 追踪导出器。

适用场景：
使用带有 `experimental_telemetry` 的 `generateText`/`streamText` 时。

必填字段：
- `OPIK_API_KEY`

可选字段：
- `OPIK_WORKSPACE`（Cloud/Enterprise 版本默认为 `default`；可设置为覆盖）
- `OPIK_PROJECT_NAME`（默认为 "Default Project"；可设置为覆盖）
- `OPIK_URL_OVERRIDE`（非默认部署 URL 时必填）
- `OPIK_LOG_LEVEL`

部署端点示例：
- Opik Cloud：`OPIK_URL_OVERRIDE=https://www.comet.com/opik/api`
- 企业版：`OPIK_URL_OVERRIDE=https://<comet-deployment-url>/opik/api`
- 自托管：`OPIK_URL_OVERRIDE=http://localhost:5173/api`

```bash filename=".env"
OPIK_API_KEY="<opik-api-key>"
OPIK_URL_OVERRIDE=https://www.comet.com/opik/api # 如果使用 Cloud 版本
OPIK_PROJECT_NAME="<custom-project-name>"
OPIK_WORKSPACE="<your-workspace>"
OPENAI_API_KEY="<openai-api-key>" # 如果使用 OpenAI 模型
```

```ts
import { OpikExporter } from "opik-vercel";

new OpikExporter();
```

接下来，通过 OpenTelemetry SDK 注册此导出器。

### Next.js

Next.js 在框架层面支持 OpenTelemetry 插桩。详情请参阅 [Next.js OpenTelemetry 指南](https://nextjs.org/docs/app/building-your-application/optimizing/open-telemetry)。

安装依赖：

```bash
npm install opik-vercel @vercel/otel @opentelemetry/api-logs @opentelemetry/instrumentation @opentelemetry/sdk-logs
```

将 `OpikExporter` 添加到插桩文件：

```ts filename="instrumentation.ts"
import { registerOTel } from "@vercel/otel";
import { OpikExporter } from "opik-vercel";

export function register() {
  registerOTel({
    serviceName: "opik-vercel-ai-nextjs-example",
    traceExporter: new OpikExporter(),
  });
}
```

### Node.js

安装依赖：

```bash
npm install opik-vercel ai @ai-sdk/openai @opentelemetry/sdk-node @opentelemetry/auto-instrumentations-node
```

```ts
import { openai } from "@ai-sdk/openai";
import { generateText } from "ai";
import { NodeSDK } from "@opentelemetry/sdk-node";
import { getNodeAutoInstrumentations } from "@opentelemetry/auto-instrumentations-node";
import { OpikExporter } from "opik-vercel";

const sdk = new NodeSDK({
  traceExporter: new OpikExporter(),
  instrumentations: [getNodeAutoInstrumentations()],
});

sdk.start();

async function main() {
  const result = await generateText({
    model: openai("gpt-4o"),
    maxTokens: 50,
    prompt: "What is love?",
    experimental_telemetry: OpikExporter.getSettings({
      name: "opik-nodejs-example",
    }),
  });

  console.log(result.text);

  await sdk.shutdown(); // 将追踪数据刷新到 Opik
}

main().catch(console.error);
```

完成！所有包含 AI SDK span 的追踪数据都会自动在 Opik 中捕获。

## AI SDK 版本与 Vercel eve

`OpikExporter` 支持所有当前的 AI SDK 版本——**v4、v5、v6 和 v7**——无需额外配置。它还支持 **Vercel eve**（基于 AI SDK v7 构建的代理框架），将每次运行捕获为嵌套的 LLM 和工具 span。上述设置即为全部所需，升级版本时代码无需修改。

### 多轮 eve 对话

运行 Vercel eve 时，多轮对话会自动归组到单个 Opik 线程中，因此每轮对话都会集中展示——你无需手动设置 `threadId`。

### 缓存 token 用量

如果你的模型报告了提示缓存，缓存 token 会与常规输入和输出 token 一起追踪，从而保持 token 和成本指标的准确性。

## 配置

### 自定义标签和元数据

你可以为 OpikExporter 生成的所有追踪添加自定义标签和元数据：

```ts
const exporter = new OpikExporter({
  // 可选：为所有追踪添加自定义标签
  tags: ["production", "gpt-4o"],
  // 可选：为所有追踪添加自定义元数据
  metadata: {
    environment: "production",
    version: "1.0.0",
    team: "ai-team",
  },
  // 可选：将追踪与对话线程关联
  threadId: "conversation-123",
});
```

标签适用于筛选和分组追踪，元数据则为调试和分析提供额外上下文。`threadId` 参数适用于追踪多轮对话或分组相关 AI 交互。

### 自定义追踪名称

```ts
const result = await generateText({
  model: openai("gpt-4o"),
  prompt: "Tell a joke",
  experimental_telemetry: OpikExporter.getSettings({
    name: "custom-trace-name",
  }),
});
```

### 线程 ID 支持

你可以通过设置 `threadId` 参数将追踪与对话线程关联。这对于追踪多轮对话或分组相关 AI 交互非常有用。

通过遥测元数据在每个请求中设置 `threadId`（这会覆盖导出器级别的 `threadId`）：

```ts
const result = await generateText({
  model: openai("gpt-4o"),
  prompt: "Continue the conversation",
  experimental_telemetry: OpikExporter.getSettings({
    name: "chat-message",
    metadata: {
      threadId: "conversation-456",
    },
  }),
});
```
