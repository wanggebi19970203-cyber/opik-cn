[Microsoft Agent Framework](https://github.com/microsoft/agent-framework) 是一个综合性多语言框架，用于构建、编排和部署 AI 智能体及多智能体工作流，同时支持 Python 和 .NET 实现。

该框架提供了从简单聊天智能体到复杂多智能体工作流的完整能力，包括基于图的编排、内置 OpenTelemetry 集成用于分布式追踪和监控，以及灵活的中间件系统用于请求/响应处理。

## 快速开始

要将 Microsoft Agent Framework .NET 集成与 Opik 一起使用，你需要安装 Agent Framework 和所需的 OpenTelemetry 包：

```bash
# Agent Framework（2 个包）
dotnet add package Microsoft.Agents.AI --prerelease
dotnet add package Microsoft.Extensions.AI.OpenAI --prerelease

# 宿主（1 个包）
dotnet add package Microsoft.Extensions.Hosting

# OpenTelemetry（3 个包）
dotnet add package OpenTelemetry.Exporter.OpenTelemetryProtocol
dotnet add package OpenTelemetry.Extensions.Hosting
dotnet add package OpenTelemetry.Instrumentation.Http
```

你还需要配置你的 OpenAI API key：

```bash
export OPENAI_API_KEY=<your-openai-api-key>
```

此外，你需要设置以下环境变量来配置 OpenTelemetry 将数据发送到 Opik：

## 在 Microsoft Agent Framework (.NET) 中使用 Opik

Microsoft Agent Framework 内置了 OpenTelemetry 检测功能。你需要配置 OpenTelemetry SDK 使用 HTTP/Protobuf 协议并导出到 Opik。以下是一个完整示例：

```csharp
using Microsoft.Agents.AI;
using Microsoft.Extensions.AI;
using Microsoft.Extensions.DependencyInjection;
using Microsoft.Extensions.Hosting;
using OpenAI;
using OpenTelemetry;
using OpenTelemetry.Resources;
using OpenTelemetry.Trace;

#pragma warning disable OPENAI001

const string SourceName = "AgentFramework";

// 创建带有 OpenTelemetry 配置的宿主
var builder = Host.CreateApplicationBuilder(args);

// 配置 OpenTelemetry
builder.Services.AddOpenTelemetry()
    .ConfigureResource(resource => resource
        .AddService(serviceName: "agent-framework-demo", serviceVersion: "1.0.0"))
    .WithTracing(tracing => tracing
        .AddSource(SourceName)                      // 我们的自定义 span
        .AddSource("*Microsoft.Extensions.AI")      // 聊天客户端遥测
        .AddSource("*Microsoft.Extensions.Agents*") // 智能体遥测
        .AddHttpClientInstrumentation()             // HTTP 调用
        .AddOtlpExporter());                        // 自动读取 OTEL_EXPORTER_OTLP_* 环境变量

var host = builder.Build();
await host.StartAsync();

// 创建带检测的聊天客户端
// 注意：OpenAI SDK 需要显式 API key，但你可以从环境变量读取
var apiKey = Environment.GetEnvironmentVariable("OPENAI_API_KEY") 
    ?? throw new InvalidOperationException("OPENAI_API_KEY 环境变量是必需的");

using var instrumentedChatClient = new OpenAIClient(apiKey)
    .GetChatClient("gpt-4o-mini")
    .AsIChatClient()                    // 转换为 Microsoft.Extensions.AI.IChatClient
    .AsBuilder()
    .UseOpenTelemetry(                  // 在聊天客户端上启用遥测
        sourceName: SourceName,
        configure: (cfg) => cfg.EnableSensitiveData = true)
    .Build();

// 创建带检测的智能体
var agent = new ChatClientAgent(
    instrumentedChatClient,
    name: "Assistant",
    instructions: "You are a helpful assistant.")
    .AsBuilder()
    .UseOpenTelemetry(SourceName)       // 在智能体上启用遥测
    .Build();

// 运行智能体 - 遥测是自动的！
Console.WriteLine(await agent.RunAsync("Write a haiku about AI agents."));

// 确保追踪被刷新
var tracerProvider = host.Services.GetService<TracerProvider>();
tracerProvider?.ForceFlush();
await Task.Delay(2000);

await host.StopAsync();
```

<Note>
**重要**：.NET OpenTelemetry SDK 需要显式设置 `OTEL_EXPORTER_OTLP_PROTOCOL=http/protobuf` 以使用 HTTP 和 Protocol Buffers。如果未设置，默认使用 gRPC，而 Opik 的端点不支持 gRPC。

你也可以通过编程方式配置：
```csharp
.AddOtlpExporter(options =>
{
    options.Protocol = OtlpExportProtocol.HttpProtobuf;
})
```
</Note>

框架将自动：
- 为智能体执行创建追踪
- 记录输入提示和输出
- 跟踪 token 使用量和性能指标
- 捕获任何错误或异常
