[Langflow](https://www.langflow.org/) 是一个开源、低代码平台，用于通过可视化的基于流程的界面构建 AI 应用程序。它使开发人员和非开发人员都能够通过拖放组件创建复杂的 LLM 应用程序，从而更轻松地进行原型设计、测试和部署 AI 工作流。

Opik 与 Langflow 无缝集成，为您的流程执行提供全面的追踪和监控。该集成自动捕获 Langflow 应用程序中的所有 LLM 调用、组件交互和数据流。

## 结果查看

配置集成后，您可以在 Opik 仪表板中查看 Langflow 追踪记录。每次流程执行都会创建一个详细的追踪，显示通过流程组件的完整执行路径。

## 快速开始

### 前提条件

在将 Opik 与 Langflow 集成之前，请确保您已具备：

- 运行中的 Langflow 服务器
- Opik Cloud 账户或自托管的 Opik 实例
- 访问运行 Langflow 的终端/环境的权限

### 安装

在同一环境中安装 Langflow 和 Opik：

```bash
pip install langflow opik
```

更多 Langflow 安装选项和详情，请参阅 [Langflow 文档](https://docs.langflow.org/)。

## 配置 Opik

配置 Opik 以连接到您的 Opik 实例。在运行 Langflow 的同一终端/环境中运行以下命令：

<Tabs>
    <Tab value="Opik Cloud" title="Opik Cloud">
        ```bash
        opik configure
        ```
        
        系统将提示您输入 Opik API 密钥。您可以在 [Opik 账户设置](https://www.comet.com/settings/api-keys?utm_source=opik&utm_medium=docs&utm_campaign=opik_integration) 中创建 API 密钥。
    </Tab>
    <Tab value="企业部署" title="企业部署">
        在运行 `opik configure` 之前设置以下环境变量：
        
        ```bash
        export OPIK_URL_OVERRIDE="https://<comet-deployment-url>/opik/api"
        opik configure
        ```
        
        系统将提示您输入 Opik API 密钥。将 `<comet-deployment-url>` 替换为您的企业部署 URL。
    </Tab>
    <Tab value="自托管实例" title="自托管实例">
        ```bash
        opik configure --use_local
        ```
        
        这将配置 Opik 使用您的本地实例 `http://localhost:5173`。
    </Tab>
</Tabs>

配置将保存到 `~/.opik.config`，并由 Langflow 的 Opik 集成自动使用。

## 在 Langflow 中使用 Opik

配置 Opik 后，在同一终端或环境中启动 Langflow：

```bash
langflow run
```

Langflow 将自动检测 Opik 配置并开始向您的 Opik 实例发送追踪记录。

### 运行流程

1. 在浏览器中打开 Langflow（通常在 `http://localhost:7860`）
2. 创建或打开一个流程
3. 点击"运行"按钮或通过 API 触发运行流程
4. 导航到 Opik 仪表板查看收集的追踪记录

### 理解 Langflow 追踪

在 Langflow 中运行流程时，Opik 会自动捕获：

- **流程执行**：整个流程执行作为一个追踪
- **组件执行**：流程中的每个组件作为独立的跨度
- **LLM 调用**：所有 LLM API 调用及其输入/输出数据
- **数据转换**：组件之间的数据传递
- **元数据**：流程配置和执行上下文

每个追踪包括：
- 流程名称和 ID
- 执行持续时间
- 输入和输出数据
- 组件级时间
- 任何错误或异常

## 高级用法

### 自定义项目名称

默认情况下，Langflow 追踪记录会记录到 Opik 中的"默认项目"。您可以在启动 Langflow 之前设置 `OPIK_PROJECT_NAME` 环境变量来指定自定义项目名称：

```bash
export OPIK_PROJECT_NAME="my-langflow-project"
langflow run
```

### 会话追踪

Langflow 使用线程 ID 自动追踪会话。每个对话或流程执行线程在 Opik 中作为独立的会话进行追踪，便于分析多轮对话和相关执行。

### 禁用集成

要禁用 Opik 集成，您可以：

1. 删除 `~/.opik.config` 文件
2. 或者取消设置 Opik 环境变量：
   ```bash
   unset OPIK_API_KEY
   unset OPIK_URL_OVERRIDE
   unset OPIK_WORKSPACE
   ```

然后重启 Langflow。

## 故障排除

### 追踪未出现在 Opik 中

如果您的追踪未出现在 Opik 中：

1. **验证 Opik 配置**：
   ```bash
   cat ~/.opik.config
   ```
   确保 API 密钥和 URL 正确。

2. **检查 Langflow 日志**：在 Langflow 控制台输出中查找任何与 Opik 相关的错误。

3. **测试 Opik 连接**：运行一个简单的 Python 脚本来验证您的 Opik 配置：
   ```python
   import opik
   client = opik.Opik()
   client.auth_check()
   print("Opik connection successful!")
   ```

4. **验证 Opik SDK 安装**：确保 Opik SDK 安装在与 Langflow 相同的环境中：
   ```bash
   pip list | grep opik
   ```

### 配置未被检测

如果 Langflow 未检测到您的 Opik 配置：

1. 确保您在启动 Langflow 的同一终端/环境中运行了 `opik configure`
2. 检查 `~/.opik.config` 文件是否存在且包含有效配置
3. 尝试显式设置环境变量：
   ```bash
   export OPIK_API_KEY="your-api-key"
   export OPIK_WORKSPACE="default"
   langflow run
   ```

### 认证错误

如果出现认证错误：

1. 在 [Opik 账户设置](https://www.comet.com/settings/api-keys?utm_source=opik&utm_medium=docs&utm_campaign=opik_integration) 中验证您的 API 密钥是否有效
2. 对于自托管实例，确保 URL 正确且实例正在运行
3. 检查您是否有权访问指定的工作空间
