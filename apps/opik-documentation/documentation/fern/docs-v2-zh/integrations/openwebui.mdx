[OpenWebUI](https://openwebui.com/) 是一个自托管的 WebUI，支持离线运行并兼容多种 LLM 运行器，包括 Ollama 和 OpenAI 兼容的 API。OpenWebUI 是开源的，可以轻松部署在您自己的基础设施上。

Opik 为 OpenWebUI 提供开源可观测性。通过 Pipelines 启用 Opik 集成，您可以使用 Opik 追踪应用数据，以开发、监控和改进 OpenWebUI 的使用，包括：

* 应用追踪
* 使用模式
* LLM 交互分析

## 如何将 Opik 与 OpenWebUI 集成

OpenWebUI 中的 Pipelines 是一个与 UI 无关的 OpenAI API 插件框架。它支持注入插件来拦截、处理和转发用户提示到最终的 LLM，从而增强对提示处理的控制和定制能力。

要使用 Opik 追踪应用数据，您可以使用 Opik 过滤管道，它支持对消息交互进行实时监控和分析。

## 设置 OpenWebUI

确保 OpenWebUI 已运行。具体操作请参考 [OpenWebUI 文档](https://docs.openwebui.com/)。

## 设置 Pipelines

使用 Docker 启动 Pipelines。运行以下命令启动 Pipelines：

```bash
docker run -p 9099:9099 --add-host=host.docker.internal:host-gateway -v pipelines:/app/pipelines --name pipelines --restart always ghcr.io/open-webui/pipelines:main
```

## 连接 OpenWebUI 与 Pipelines

在 **Admin Panel > Settings** 中，创建并保存一个新的 OpenAI API 类型连接，配置如下：

* **URL：** `http://localhost:9099/`（这是之前启动的 Docker 容器的运行地址）
* **密码：** `0p3n-w3bu!`（默认密码）

<Note>
如果您的 Open WebUI 运行在 Docker 容器中，请将 API URL 中的 `localhost` 替换为 `host.docker.internal`。
</Note>

![OpenWebUI 设置](/img/tracing/openwebui_settings.png)

## 添加 Opik 过滤管道

接下来，导航到 **Admin Panel > Settings > Pipelines**，添加 Opik 过滤管道。配置管道连接时，使用与上一步相同的 URL（如果 OpenWebUI 在 Docker 中运行，则为 `http://localhost:9099` 或 `http://host.docker.internal:9099`）。通过 _Install from GitHub URL_ 选项安装 Opik 过滤管道，URL 如下：

```
https://github.com/open-webui/pipelines/blob/main/examples/filters/opik_filter_pipeline.py
```

然后在下方添加您的 Opik API 密钥。如果您还没有注册 Opik，可以在[此处](https://www.comet.com/signup?from=opik)创建账号获取 API 密钥。

![OpenWebUI 添加 Opik 管道](/img/tracing/openwebui_add_pipeline.png)

### 配置选项

安装管道后，您可以在 **Pipelines Valves** 部分配置以下选项：

* **Api Key**：您的 Opik API 密钥（必填）
* **Workspace**：您的 Opik 工作区名称（可选，默认为 "default"）
* **Project Name**：用于组织追踪的项目名称（可选）
* **Host**：您的 Opik 实例 URL（Opik Cloud 默认为 `https://www.comet.com/opik/api`）
* **Debug**：启用调试日志（可选）

### 捕获 Token 使用量

要捕获模型的 token 使用数据，请导航到 OpenWebUI 的模型设置，在 _Capabilities_ 下勾选 **"Usage"** 复选框。
