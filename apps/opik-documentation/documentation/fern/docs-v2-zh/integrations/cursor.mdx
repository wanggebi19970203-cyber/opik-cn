## 安装

你也可以通过导航到文件侧边栏顶部的 `Extensions` 标签页手动安装 Cursor 扩展，搜索 `Opik`，然后点击 `Install` 按钮即可。

<Button intent="primary" href="cursor:extension/opik.opik">
  安装 Cursor 扩展
</Button>

### 配置扩展

要使用该扩展，你需要配置 Opik API 密钥。有以下几种方式：

<Tabs>
    <Tab title="扩展设置">
    配置扩展最直接的方式是使用扩展设置。

    安装扩展后，屏幕左下角会显示以下弹窗：
    <img src="/img/tracing/cursor_opik_extension_settings.png" />

    在该弹窗中，点击 `Open Settings` 并将你的 Opik API 密钥粘贴到 `Opik: Opik API Key` 字段中。
    你可以在 [Opik 控制面板](https://www.comet.com/api/my/settings) 中找到你的 API 密钥。

    </Tab>
    <Tab title="配置文件">
    你也可以在主目录中使用配置文件（`~/.opik.config`）。虽然你可以手动创建该文件（参考文档请见[这里](/tracing/advanced/sdk_configuration)），但如果你已安装 Opik Python SDK，我们建议使用 `opik configure` 命令来自动创建该文件。
    </Tab>

</Tabs>

扩展安装完成后，你所有的新 Cursor 对话都将被记录到 Opik 中。

## MCP 服务器集成

当你配置了 API 密钥后，扩展会自动向 Cursor 注册一个 Opik MCP（模型上下文协议）服务器。这使得 Cursor 的 AI 在对话过程中可以访问你的 Opik 项目数据，为你的追踪和实验提供增强的上下文信息。

MCP 服务器集成默认启用，无需额外配置。有关 MCP 服务器的更多详情，请参阅 [opik-mcp 包](https://www.npmjs.com/package/opik-mcp)。

## 查看对话

扩展安装并配置完成后，你所有的对话都将存储在 Opik 的 `cursor` 项目中。要查看完整对话，可以导航到 `threads` 标签页并点击任意聊天记录。

你也可以通过点击 `traces` 标签页中的追踪记录来查看单个 LLM 响应。在追踪侧边栏视图中，你可以在 metadata 部分查看完整的 Cursor 事件数据。
