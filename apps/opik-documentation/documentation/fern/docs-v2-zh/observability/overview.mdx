LLM 应用不仅仅是单个 API 调用。一个典型的智能体涉及检索步骤、工具调用、提示组装、多次 LLM 调用和后处理——所有这些在运行时都是不可见的。当出现问题时，您需要查看每个步骤具体发生了什么。

Opik 为您的智能体处理的每个请求提供完整可见性。每次 LLM 调用、每次工具调用、每个检索步骤都被捕获为可检查、搜索和分析的追踪记录。

## 为什么使用 Opik 进行可观测性

没有可观测性的 LLM 应用调试意味着猜测。您看到最终输出，但不知道模型为何产生幻觉、哪个检索步骤返回了不相关的上下文，或延迟在哪里飙升。

使用 Opik，您可以：

- **查看每个请求的完整执行路径**——从用户输入到工具调用和 LLM 完成，再到最终响应
- **快速定位生产问题**——按状态、延迟、成本或自定义标签过滤和搜索追踪记录，秒级定位问题
- **跟踪成本和延迟趋势**——监控跨模型和提供商的 Token 使用量和支出
- **捕获多轮对话**——将相关追踪记录分组到线程中，了解交互如何在各轮之间演进
- **闭环反馈**——为追踪记录附加人工或自动化评分，并将其用于驱动评估

## 工作原理

<Steps>
  <Step title="连接您的项目">
    从智能体目录运行 `opik connect` 将其与 Opik 配对：

    ```bash
    opik connect --project <YOUR_PROJECT_NAME>
    ```
  </Step>
  <Step title="为代码添加追踪">
    添加追踪的最快方式是使用 [opik-skills](https://github.com/comet-ml/opik-skills)——安装该技能，让您的编码智能体处理其余工作：

    ```bash
    npx skills add comet-ml/opik-skills
    ```

    然后询问您的编码智能体：

    ```
    使用 /instrument 命令为我的智能体添加 Opik 追踪。
    ```

    这适用于 Claude Code、Cursor、Codex、OpenCode 和其他编码智能体。您也可以使用 SDK 手动添加追踪：

    <CodeBlocks>
      ```python title="Python"
      import opik

      @opik.track
      def my_agent(user_message):
          context = retrieve_context(user_message)
          response = call_llm(user_message, context)
          return response
      ```

      ```ts title="TypeScript"
      import { Opik } from "opik";
      const client = new Opik();

      const myAgent = client.track({
        name: "my-agent",
        fn: async (userMessage: string) => {
          const context = await retrieveContext(userMessage);
          return await callLlm(userMessage, context);
        },
      });
      ```
    </CodeBlocks>
  </Step>
  <Step title="在仪表板中查看追踪记录">
    每个请求都会创建包含详细跨度级信息的追踪记录。您可以检查完整的执行树、查看每个步骤的输入和输出，并按持续时间、成本、状态或标签进行过滤。

    <Frame>
      <img src="/img/v2/home/traces_page_for_quickstart.png" />
    </Frame>
  </Step>
  <Step title="分析与改进">
    使用追踪记录调试故障、识别慢速步骤并跟踪质量趋势。附加反馈评分、对数据集运行评估，并使用 [Ollie](/tracing/debug-agents)（Opik 的 AI 助手）自动帮助定位问题根源。
  </Step>
</Steps>
