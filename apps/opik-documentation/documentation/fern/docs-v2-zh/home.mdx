Opik 是一个[开源](https://github.com/comet-ml/opik)平台，帮助你理解 LLM 应用的运行状况，衡量其表现，并系统性地进行优化。无论你正在构建聊天机器人、RAG 管道还是多步骤智能体，Opik 都能提供工具，帮助你从"在我的笔记本上能运行"进阶到"在生产环境中稳定运行"。

## 如何使用 Opik

<Tip>
  **使用 Claude Code、Cursor 或 VS Code Copilot？** 安装 [Opik MCP 服务器](/mcp-server)，即可在聊天中驱动整个工作区 — 读取追踪记录、评分输出、保存提示词并运行实验，无需打开 UI。
</Tip>

<Steps>
  ### 查看应用运行状态

  Opik 记录每次 LLM 调用、工具调用和智能体步骤，让你能够检查导致任何输出的完整事件链。只需添加几行代码，即可获得每个请求和响应的完整日志。

  [开始记录追踪 →](/tracing/getting-started)

  ### 从追踪记录构建测试套件

  当你发现异常的追踪记录时，将其转化为测试用例。使用 [Ollie](/ollie) 自动完成（只需描述问题所在），或通过 UI 或 SDK 添加测试用例。然后使用 Ollie 或 SDK 运行测试套件以验证修复效果。

  随着时间推移，你的测试套件将基于真实的生产故障不断增长，而非假设性示例。

  [构建你的第一个测试套件 →](/evaluation/advanced/building-test-suites)

  ### 跟踪生产环境质量

  设置在线评估规则以自动评分传入的追踪记录，并从项目仪表板监控反馈分数、延迟、成本和错误率。

  [设置生产监控 →](/production/online-evaluation/rules)

  ### 自动优化提示词

  Opik 的优化算法会根据你的指标和数据集测试提示词的变体，找到最佳方案，无需手动试错。

  [运行你的第一次优化 →](/development/optimization-runs/quickstart)
</Steps>
