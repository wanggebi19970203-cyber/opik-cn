Opik 是[开源的](https://github.com/comet-ml/opik)，并且正在非常积极地开发中。我们使用 Opik 社区的反馈来驱动路线图，这是一份非常活跃的文档，将随着我们发布新功能和了解改进产品的新方式而变化。

<Tip>
如果您对路线图有任何想法或建议，可以在 Opik Github 仓库中创建[新的功能请求 issue](https://github.com/comet-ml/opik/issues/new/choose)。
</Tip>

## 🚀 我们最近发布了什么？
好奇我们的最新更新？在我们的[更新日志](/v1/changelog)中探索我们发布的一切。

## 🛠️ 我们目前正在做什么？

我们目前正在改进现有功能和开发新功能：

**代理和优化**：
  - 代理和提示优化的新功能

**战略工作**：
  - 高级多模态支持
  - 新的 Webhook 警报系统

**开发者体验**：
  - JS/TS SDK 和 REST API 改进支持

**用户体验**：
  - 改进的入门体验
  - 更新的文档和移动体验


## 🧭 接下来计划什么？

**代理和优化**：
  - 简化的优化工作流：新的优化工作室。
  - 构建更好的 AI：由新算法驱动的高级代理优化。
  - 支持延迟和成本的多目标优化器

**用户体验**：
  - 增强的表格支持：更智能的排序、灵活的过滤、无缝导出和强大的批量操作，实现轻松的数据管理。


## 🔭 未来计划是什么？

展望更远的未来，我们计划在更深层次的优化、协作和可观测性工具方面进行投资：

**高级评估**：
  - 更智能的 AI 性能评估：用于 LLM 作为评判者指标的人类反馈强化学习。

**闭环改进**：
  - 通过利用生产数据和关键性能指标，自动优化整个生命周期的代理

**项目和数据集改进**：
  - 数据集编辑和版本控制
  - 支持公共和私有项目
  - 在 Opik 游乐场中引入评估功能

**监控和分析**：
  - 引入仪表板功能


## 💬 提供您的反馈

我们依靠您的反馈来塑造路线图并决定接下来优先开发哪些功能。您可以在 [Github Issues](https://github.com/comet-ml/opik/issues/) 上对现有想法投票或添加自己的想法。

*最后更新：2025 年第四季度*
