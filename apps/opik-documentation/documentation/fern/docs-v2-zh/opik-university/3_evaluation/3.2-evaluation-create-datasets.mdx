## 为 RAG 评估构建数据集

本实践视频演示使用实际 RAG（检索增强生成）示例创建[数据集](/v1/evaluation/manage_datasets)，比较 [OpenAI](/v1/integrations/openai) 和 Google [Gemini](/v1/integrations/gemini) 模型。您将了解[评估数据集](/v1/evaluation/manage_datasets)如何作为系统化 LLM 测试的基础 - 它们是应用将遇到的示例输入集合以及预期输出，类似于传统机器学习中的验证集。

## 关键亮点

- **实际 RAG 设置**：完整示例展示 [OpenAI](/v1/integrations/openai) vs Google [Gemini](/v1/integrations/gemini) 模型比较，使用 [LangChain](/v1/integrations/langchain) 和 Chroma 进行向量存储集成
- **双重创建方法**：通过 UI（选择[追踪](/v1/tracing/log_traces)并"添加到数据集"）或使用 Opik 客户端的 [`get_or_create_dataset()`](/v1/evaluation/manage_datasets#creating-a-dataset-using-the-sdk) 编程创建[数据集](/v1/evaluation/manage_datasets)
- **灵活的数据结构**：根据您的特定用例在[数据集](/v1/evaluation/manage_datasets)项目中定义自定义字段 - 使输入和输出与应用需求一样详细
- **自动去重**：Opik 自动防止[数据集](/v1/evaluation/manage_datasets)中的重复条目，确保数据质量和一致性
- **多[数据集](/v1/evaluation/manage_datasets)策略**：为不同方面创建专注的[数据集](/v1/evaluation/manage_datasets) - 常见问题、边缘情况、失败模式以及推理或总结等特定能力
- **追踪到[数据集](/v1/evaluation/manage_datasets)转换**：通过过滤高性能交互并将其直接转换为[评估数据集](/v1/evaluation/manage_datasets)来利用现有[追踪](/v1/tracing/log_traces)
- **验证集方法**：[数据集](/v1/evaluation/manage_datasets)的功能类似于传统 ML 验证集，为系统化性能评估提供代表性示例
- **可扩展架构**：使用基于类的设置处理不同模型提供商，同时使用 [`@track`](/v1/tracing/log_traces#decorators) 装饰器维护可追溯性
