## 使用 Opik 游乐场进行交互式提示工程

本视频演示如何使用 Opik 的[提示游乐场](/v1/prompt_engineering/playground)进行实践[提示工程](/v1/prompt_engineering/prompt_management)和测试。[游乐场](/v1/prompt_engineering/playground)提供了一个交互式环境，您可以在其中制作[提示](/v1/prompt_engineering/prompt_management)、测试变体，并直接在 UI 中运行[实验](/v1/evaluation/evaluate_your_llm)。您将学习如何利用库中的版本化[提示](/v1/prompt_engineering/prompt_management)、进行 A/B 测试，以及从提示开发无缝过渡到完整的[数据集评估](/v1/evaluation/manage_datasets)。

## 关键亮点

- **并行 A/B 测试**：并排运行多个[提示](/v1/prompt_engineering/prompt_management)变体以比较输出，为您的用例识别最有效的方法
- **灵活的[提示](/v1/prompt_engineering/prompt_management)类型**：根据工作流程需求在系统和用户[提示](/v1/prompt_engineering/prompt_management)之间切换，从简单查询到复杂系统指令
- **版本化[提示](/v1/prompt_engineering/prompt_management)集成**：直接从托管的[提示库](/v1/prompt_engineering/prompt_management)加载[提示](/v1/prompt_engineering/prompt_management)，始终访问最新提交的版本以进行一致测试
- **[数据集](/v1/evaluation/manage_datasets)驱动的[实验](/v1/evaluation/evaluate_your_llm)**：将测试[数据集](/v1/evaluation/manage_datasets)直接集成到[游乐场](/v1/prompt_engineering/playground)会话中，直接从界面运行全面[评估](/v1/evaluation/overview)
- **可配置的模型参数**：调整创造力级别、输出 token 限制和其他模型设置，以根据您的特定需求微调行为
- **模板变量支持**：使用 mustache/handlebar 表示法（例如 `{{question}}`）创建与[数据集](/v1/evaluation/manage_datasets)变量配合使用的动态[提示](/v1/prompt_engineering/prompt_management)
- **实时版本控制**：直接从[游乐场](/v1/prompt_engineering/playground)保存[提示](/v1/prompt_engineering/prompt_management)修改，带有提交消息和元数据，在[提示库](/v1/prompt_engineering/prompt_management)中创建新版本
- **即时[实验](/v1/evaluation/evaluate_your_llm)创建**：生成带有可视反馈横幅的[实验](/v1/evaluation/evaluate_your_llm)，然后查看包括问答[数据集](/v1/evaluation/manage_datasets)输出在内的全面结果
- **无缝工作流集成**：在不离开 Opik 环境的情况下，弥合[提示](/v1/prompt_engineering/prompt_management)开发、测试和完整[评估](/v1/evaluation/overview)之间的差距
