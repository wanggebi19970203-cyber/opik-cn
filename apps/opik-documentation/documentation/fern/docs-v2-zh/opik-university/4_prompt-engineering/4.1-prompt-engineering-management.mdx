## 通过系统化管理扩展提示工程

本视频讨论 LLM 应用开发中一个关键但经常被忽视的方面：[提示管理](/v1/prompt_engineering/prompt_management)。随着应用的增长，团队可能会有数十或数百个[提示](/v1/prompt_engineering/prompt_management)，如果没有适当的组织，这些提示会变得重复、过时或不一致地应用。您将了解 Opik 的[提示管理](/v1/prompt_engineering/prompt_management)功能如何帮助组织、版本化和在团队间共享[提示](/v1/prompt_engineering/prompt_management)，将混乱的提示处理转变为系统化、可扩展的工作流程。

## 关键亮点

- **系统化组织**：将[提示](/v1/prompt_engineering/prompt_management)组织到逻辑"提示桶"中，使用清晰的命名约定（基于项目 + 提示类型）以防止规模扩大时的混乱
- **完整版本历史**：跟踪完整的提交历史，每个[提示](/v1/prompt_engineering/prompt_management)有 16+ 个版本，包括时间戳和提交消息，提供完整的提示演进可见性
- **[实验](/v1/evaluation/evaluate_your_llm)集成**：查看连接到特定[提示](/v1/prompt_engineering/prompt_management)的所有[实验](/v1/evaluation/evaluate_your_llm)，轻松识别哪些提示变体在不同测试运行中表现最佳
- **并排比较**：使用比较功能分析[提示](/v1/prompt_engineering/prompt_management)版本之间的差异，实现数据驱动的提示优化决策
- **丰富的元数据支持**：将键值元数据作为标准 JSON 对象附加到[提示](/v1/prompt_engineering/prompt_management)，提供额外的上下文和组织
- **编程管理**：使用 [`opik.Prompt`](/v1/prompt_engineering/prompt_management) 类直接从代码创建和提交[提示](/v1/prompt_engineering/prompt_management)，将提示管理集成到开发工作流程中
- **团队协作**：在团队成员之间标记和共享[提示](/v1/prompt_engineering/prompt_management)，确保一致性并防止项目间的重复
- **性能跟踪**：直接在提示视图中查看来自[实验](/v1/evaluation/evaluate_your_llm)的附加[指标](/v1/evaluation/metrics/overview)，将提示更改与性能结果联系起来
- **[游乐场](/v1/prompt_engineering/playground)集成**：在 Opik 的[游乐场](/v1/prompt_engineering/playground)中无缝使用跟踪的[提示](/v1/prompt_engineering/prompt_management)进行测试和迭代（在下一个视频中介绍）
